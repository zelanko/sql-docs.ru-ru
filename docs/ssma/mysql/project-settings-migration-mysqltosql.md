# <a name="project-settings-migration-mysqltosql"></a>Параметры проекта (миграция) (MySQLToSQL)
Страница миграция диалогового окна **Параметры проекта** содержит параметры, которые позволяют настроить SSMA миграцию данных из MySQL в SQL Server.  
  
Панель миграция доступна в диалоговых окнах **Параметры проекта** и **Параметры проекта по умолчанию** .  
  
-   Чтобы задать параметры для всех проектов SSMA, в меню **Сервис** выберите **Параметры проекта по умолчанию**, выберите в раскрывающемся списке **версия целевого объекта миграции** тип проекта, для которого требуется получить доступ к параметрам, а затем щелкните **Общие** в нижней части левой панели и выберите **Миграция**.  
  
-   Чтобы указать параметры для текущего проекта, в меню **Сервис** выберите **Параметры проекта**, щелкните **Общие** в нижней части левой панели, а затем щелкните **Миграция**.  
  
## <a name="options"></a>Параметры  
  
### <a name="bulk-copy"></a>Массовое копирование  
  
|Термин|Определение|  
|--------|--------------|  
|**Размер пакета**|Указывает размер пакета, используемый при переносе данных.<br /><br />**Режим по умолчанию**: 1000<br /><br />**Оптимистичный режим**: 1000<br /><br />**Полный режим**: 1000|  
|**Проверочные ограничения**|Указывает, должны ли SSMA проверять ограничения при вставке данных в SQL Server таблицы.<br /><br />**Режим по умолчанию**: false<br /><br />**Оптимистичный режим**: false<br /><br />**Полный режим**: false|  
|**Пожарные триггеры**|Указывает, должен ли SSMA запускать триггеры вставки при добавлении данных в SQL Server таблицы.<br /><br />**Режим по умолчанию**: false<br /><br />**Оптимистичный режим**: false<br /><br />**Полный режим**: false|  
|**Сохранит удостоверение**|Указывает, сохраняет ли SSMA значения идентификаторов MySQL при добавлении данных в SQL Server. Значение false приводит к назначению значений идентификаторов целевым объектом.<br /><br />**Режим по умолчанию**: true<br /><br />**Оптимистичный режим**: true<br /><br />**Полный режим**: true|  
|**Сохранения значений NULL**|Указывает, сохраняет ли SSMA значения NULL в исходных данных при добавлении данных в SQL Server, независимо от значений по умолчанию, указанных в SQL Server.<br /><br />**Режим по умолчанию**: true<br /><br />**Оптимистичный режим**: true<br /><br />**Полный режим**: true|  
|**Блокировка таблицы**|Указывает, следует ли SSMA блокировать таблицы при добавлении данных в таблицы во время переноса данных. Получает блокировку с массовым обновлением в течение операции с массовым копированием. Если значение равно false, блокировка задается на уровне строк.<br /><br />**Режим по умолчанию**: false<br /><br />**Оптимистичный режим**: false<br /><br />**Полный режим**: false|  
  
### <a name="data-modification"></a>Изменение данных  
  
|Термин|Определение|  
|--------|--------------|  
|**Недопустимая миграция дат**|Указывает, как переносить недопустимые даты с помощью значений "2007-04-23" или "2000-06-31 10:00:00" в форматах даты и времени.<br /><br />**Режим по умолчанию**: задать значение null<br /><br />**Оптимистичный режим**: задать значение null<br /><br />**Полный режим**: задать значение null|  
|**Миграция отрицательных значений времени**|Указывает способ переноса отрицательных значений, например "-30:11:00", в столбцы времени.<br /><br />**Режим по умолчанию**: задать значение null<br /><br />**Оптимистичный режим**: задать значение null<br /><br />**Полный режим**: задать значение null|  
|**Значения времени за период с миграцией за 24 часа**|Указывает, как перенести значения времени, превышающие "23:59:59", в столбцы времени.<br /><br />**Режим по умолчанию**: задать значение null<br /><br />**Оптимистичный режим**: задать значение null<br /><br />**Полный режим**: задать значение null|  
|**Усечение двоичных значений по размеру столбца**|Если да, SSMA усекает двоичные значения из MySQL, которые не помещаются в столбцы таблицы SQL, и создает соответствующее сообщение об ошибке. Если нет, строка вызывает ошибку.<br /><br />**Режим по умолчанию**: нет<br /><br />**Оптимистичный режим**: нет<br /><br />**Полный режим**: нет|  
|**Усечение значений символов в соответствии со столбцом**|SSMA усекает символьные значения из MySQL, которые не помещаются в столбцы таблицы SQL, и создает соответствующее сообщение об ошибке.<br /><br />**Режим по умолчанию**: нет<br /><br />**Оптимистичный режим**: нет<br /><br />**Полный режим**: нет|  
|**Миграция нулевых дат**|Указывает, как выполнять миграцию нулевых дат, таких как "0000-00-00" или "0000-00-00 00:00:00", в столбцах DATE и DATETIME.<br /><br />**Режим по умолчанию**: задать значение null<br /><br />**Оптимистичный режим**: задать значение null<br /><br />**Полный режим**: задать значение null|  
|**Миграция нулей в датах**|Указывает, как выполнять миграцию дат с нулевыми частями, такими как "2009-01-00" или "2000-00-00 11:00:00", в столбцах DATE и DATETIME.<br /><br />**Режим по умолчанию**: задать значение null<br /><br />**Оптимистичный режим**: задать значение null<br /><br />**Полный режим**: задать значение null|  
  
### <a name="migration-engine"></a>Модуль миграции  
  
|Термин|Определение|  
|--------|--------------|  
|**Модуль миграции**|Указывает ядро СУБД, используемое при переносе данных. Перенос данных на стороне клиента относится к клиенту SSMA, получая данные из источника и добавляя их в SQL Server. Миграция данных на стороне сервера относится к подсистеме миграции данных SSMA (программа по копированию), работающей на SQL Server, в качестве задания агента SQL, получая данные из источника и вставляя их непосредственно в SQL Server тем самым избегая дополнительного прыжка клиента (более высокая производительность).<br /><br />**Режим по умолчанию**: модуль миграции данных на стороне клиента<br /><br />**Оптимистичный режим**: модуль миграции данных на стороне клиента<br /><br />**Полный режим**: модуль миграции данных на стороне клиента|  
  
> [!IMPORTANT]  
> Если для параметра **модуль миграции** установлено значение **модуль миграции данных на стороне сервера**, то отображается новый параметр проекта **использовать ядро миграции данных на стороне сервера 32-bit Server** . Он указывает, используется ли служебная программа для переноса данных в 32-разрядной или 64 разрядной программе (BCP).  
  
### <a name="misc"></a>Разное  
  
|Термин|Определение|  
|--------|--------------|  
|**Параметры расширенной миграции данных**|Отображает дополнительные параметры переноса данных для каждой таблицы на отдельной вкладке сведений.<br /><br />**Режим по умолчанию**: скрыть<br /><br />**Оптимистичный режим**: скрытие<br /><br />**Полный режим**: скрыть|  
|**При ошибке**|Останавливает перенос данных при возникновении ошибки. Он имеет три варианта:<br /><br />**Прерывать миграцию:** Останавливает операцию миграции данных<br /><br />**Перейти к следующей таблице:** Останавливает миграцию данных в текущую таблицу и переходит к следующей<br /><br />**Перейти к следующему пакету:** Останавливает миграцию данных в текущий пакет и переходит к следующему<br /><br />**Режим по умолчанию**: перейти к следующему пакету<br /><br />**Оптимистичный режим**: переход к следующему пакету<br /><br />**Полный режим**: переход к следующему пакету|  
  
### <a name="parallel-data-migration"></a>Параллельная миграция данных  
  
|Термин|Определение|  
|--------|--------------|  
|**Режим параллельной миграции данных**|Указывает режим, используемый для разветвления потоков, чтобы включить параллельную миграцию данных. В режиме auto SSMA выбирает число потоков (по умолчанию — 10), разветвленных для переноса данных. В пользовательском режиме пользователь может указать число потоков, разделенных для переноса данных (минимальное значение — 1, а максимальное — 100). Сейчас только модуль миграции данных на стороне клиента поддерживает параллельную миграцию данных.<br /><br />**Режим по умолчанию**: Авто<br /><br />**Оптимистичный режим**: автоматический<br /><br />**Полный режим**: Авто|  
  
> [!IMPORTANT]  
> Если для параметра **режим параллельной миграции данных** задано значение **Настраиваемый**, то отображается новый параметр проекта параметр **число потоков** . Указывает число потоков, используемых для переноса данных.  
  
### <a name="spatial-data"></a>Пространственные данные  
  
|Термин|Определение|  
|--------|--------------|  
|**Обработка ошибок**|Указывает способ обработки ошибок при переносе значений пространственных типов данных. Если указан параметр "Replace with NULL", все пространственные значения, вызывающие ошибки, будут заменены на NULL. В противном случае замена не выполняется.<br /><br />**Режим по умолчанию**: Создание ошибки<br /><br />**Оптимистичный режим**: Создание ошибки<br /><br />**Полный режим**: Создание ошибки|  
|**Проверка значения**|Указывает способ обработки недопустимых пространственных значений. Если указана попытка "попробовать сделать допустимой", предпринимается попытка изменить недопустимые значения, чтобы сделать их допустимыми.<br /><br />**Режим по умолчанию**: сделать допустимым<br /><br />**Оптимистичный режим**: не изменять<br /><br />**Полный режим**: сделать допустимым|
