# <a name="getting-started-with-ssma-for-mysql-mysqltosql"></a>Начало работы с SSMA для MySQL (MySQLToSQL)
SQL Server Migration Assistant (SSMA) для MySQL позволяет быстро преобразование схем баз данных MySQL в схемы SQL Server или база данных SQL Azure, передача результирующей схемы в SQL Server или в базе данных SQL Azure и переноса данных из MySQL в SQL Server или база данных SQL Azure.  
  
В этом разделе представлен процесс установки, а затем ознакомиться с SSMA пользовательского интерфейса.  
  
## <a name="installing-ssma"></a>Установка SSMA  
Чтобы использовать SSMA, сначала необходимо установить SSMA клиентской программы на компьютере, который можно получить доступ к базе данных MySQL исходного и целевого экземпляра SQL Server или база данных SQL Azure. Установите MySQL поставщиков (MySQL 5.1 драйвера ODBC (доверия)) на компьютере, на котором выполняется клиентская программа SSMA. Инструкции по установке см. в разделе [Установка SSMA для MySQL &#40; MySqlToSql &#41;](../../ssma/mysql/installing-ssma-for-mysql-mysqltosql.md)  
  
Для запуска SSMA нажмите кнопку **запустить**, пункты **все программы**, пункты **SQL Server Migration Assistant для MySQL**, а затем нажмите кнопку **SQL Server Migration Assistant для MySQL**.  
  
## <a name="ssma-for-mysql-user-interface"></a>SSMA для интерфейса пользователя MySQL  
После SSMA установлен и лицензирован, можно использовать SSMA для переноса баз данных MySQL в SQL Server или база данных SQL Azure. Это позволяет ознакомиться с пользовательским интерфейсом SSMA перед запуском. На следующей диаграмме показан пользовательский интерфейс для SSMA, включая метаданные обозревателей, метаданных, панели инструментов, области вывода и панели «Список ошибок»:  
  
![SSMA для графического интерфейса пользователя MySql](../../ssma/mysql/media/ssmaformysqlgui.gif "SSMA для графического интерфейса пользователя MySql")  
  
Чтобы начать миграцию, необходимо выполнить следующее:  
  
1.  Создайте новый проект.  
  
2.  Подключиться к базе данных MySQL.  
  
3.  После успешного подключения схемы MySQL будет отображаться в обозревателе метаданных MySQL. Объекты правой кнопкой мыши в обозревателе метаданных MySQL для выполнения задач, таких как создание отчетов, оценить преобразования к базе данных SQL Server/Azure SQL.  
  
Эти задачи можно также выполнить с помощью панели инструментов и меню.  
  
Кроме того, необходимо подключиться к экземпляру SQL Server. После успешного подключения иерархии баз данных SQL Server будет отображаться в обозревателе метаданных SQL Server. После преобразования схемы MySQL в схемы SQL Server, выберите преобразованные схемы в обозревателе метаданных SQL Server и затем выполните синхронизацию схемы с SQL Server.  
  
Если выбрана база данных SQL Azure из миграции раскрывающийся список в диалоговое окно нового проекта необходимо подключиться к базе данных SQL Azure. После успешного подключения иерархии баз данных в базе данных SQL Azure будет отображаться в обозревателе метаданных базы данных SQL Azure. После преобразования схемы MySQL в схемы базы данных SQL Azure, выберите преобразованные схемы в обозревателе метаданных базы данных SQL Azure, а затем синхронизировать схемы с базу данных SQL Azure.  
  
После синхронизации преобразованные схемы с SQL Server или база данных SQL Azure, можно вернуться в обозреватель метаданных MySQL и переноса данных из MySQL схем в базы данных SQL Server или база данных SQL Azure.  
  
Дополнительные сведения об этих задачах и порядок их выполнения см. в разделе [миграции баз данных MySQL в SQL Server — база данных SQL Azure &#40; MySQLToSql &#41; ](../../ssma/mysql/migrating-mysql-databases-to-sql-server-azure-sql-db-mysqltosql.md).  
  
В следующих разделах описаны возможности SSMA пользовательского интерфейса.  
  
### <a name="metadata-explorers"></a>Обозреватели метаданных  
SSMA содержит два обозреватели метаданных для просмотра и выполнять действия от баз данных MySQL и SQL Server.  
  
### <a name="mysql-metadata-explorer"></a>Обозреватель метаданных MySQL  
Обозреватель метаданных MySQL показывает сведения о схемах MySQL. С помощью обозревателя метаданных MySQL, можно выполнять следующие задачи:  
  
-   Просматривать объекты в каждой схеме.  
  
-   Выберите объекты для преобразования и затем преобразуйте эти объекты в синтаксис SQL Server. Дополнительные сведения см. в разделе [преобразование баз данных MySQL &#40; MySQLToSQL &#41;](../../ssma/mysql/converting-mysql-databases-mysqltosql.md)  
  
-   Выберите таблицы для переноса данных, а затем перенести данные из этих таблиц в SQL Server. Дополнительные сведения см. в разделе [переноса данных MySQL в SQL Server — база данных SQL Azure &#40; MySQLToSQL &#41;](../../ssma/mysql/migrating-mysql-data-into-sql-server-azure-sql-db-mysqltosql.md)  
  
### <a name="sql-server-or-azure-sql-db-metadata-explorer"></a>SQL Server или в обозревателе метаданных базы данных Azure SQL  
SQL Server или базы данных SQL Azure в обозревателе метаданных содержит сведения об экземпляре SQL Server или база данных SQL Azure. При подключении к экземпляру SQL Server или база данных SQL Azure SSMA извлекает метаданные об этом экземпляре и сохраняет его в файле проекта.  
  
Позволяет выбрать преобразованные объекты базы данных MySQL в обозревателе метаданных и затем синхронизировать их с экземпляром SQL Server или база данных SQL Azure.  
  
Дополнительные сведения см. в разделе [синхронизации (MySQL в SQL Server и базу данных SQL Azure)](http://msdn.microsoft.com/en-us/ac993a6d-0283-4823-8793-6b217677dfa3)  
  
### <a name="metadata"></a>Метаданные  
Справа от каждого обозреватель метаданных являются вкладки, которые описывают выбранного объекта. Например, если выбрать таблицу в обозревателе метаданных MySQL, девять вкладки располагаются: **таблицы**, **SQL**, **сопоставление типов**, **данные**, **параметры**, **сопоставление Charset**, **режимах SQL**, **свойства**, и **отчета**. **Отчетов** вкладка содержит сведения только после создания отчета, содержащий выбранный объект. Если выбрать таблицу в обозревателе метаданных SQL Server, будут отображаться три вкладки: **таблицы**, **SQL** и **данные**.  
  
Большинство параметров метаданные доступны только для чтения. Тем не менее можно изменить следующие метаданные:  
  
-   В обозревателе метаданных MySQL может изменять сопоставления типов, сопоставление Charset, режимы SQL. Чтобы преобразовать измененный тип сопоставления или сопоставление Charset или режимы SQL, внесите изменения, перед началом преобразования схемы.  
  
-   В обозревателе метаданных SQL Server можно изменить свойства таблиц и индексов на вкладке таблицы. Чтобы увидеть эти изменения в SQL Server, внести эти изменения, прежде чем загружать эти схемы в SQL Server.  
  
Изменения, внесенные в обозревателе метаданных, отражаются в метаданные проекта, не в исходной или целевой базы данных.  
  
### <a name="toolbars"></a>Панели инструментов  
SSMA имеет две панели: панель инструментов проекта и инструментов миграции.  
  
### <a name="the-project-toolbar"></a>Панели инструментов проекта  
Панель инструментов проекта содержит кнопки для работы с проектами, подключение к MySQL и подключение к SQL Server или база данных SQL Azure. Эти кнопки команды похожи на **файл** меню.  
  
### <a name="migration-toolbar"></a>Панель инструментов миграции  
В следующей таблице показаны миграции команды панели инструментов:  
  
|||  
|-|-|  
|**Кнопка**|**Компонент**|  
|**Создание отчета**|Преобразует выбранные объекты MySQL объектов SQL Server или база данных SQL Azure и создает отчет, которое показывает успешность преобразования.<br /><br />Эта команда недоступна, если только выбранных объектов в обозревателе метаданных MySQL.|  
|**Преобразовать схему**|Преобразует выбранные объекты MySQL объектов SQL Server или база данных SQL Azure.<br /><br />Эта команда недоступна, если только выбранных объектов в обозревателе метаданных MySQL.|  
|**Перенос данных**|Перемещает данные из базы данных MySQL в SQL Server или база данных SQL Azure. Перед выполнением этой команды необходимо преобразовать схем MySQL в схемы SQL Server или база данных SQL Azure и затем загрузить объекты в SQL Server или база данных SQL Azure.<br /><br />Эта команда недоступна, если только выбранных объектов в обозревателе метаданных MySQL.|  
|**Остановить**|Останавливает текущий процесс.|

В следующей таблице показаны SSMA меню.  
  
|||  
|-|-|  
|**Меню**|**Description**|  
|**Файл**|Команды для работы с проектами, подключение к MySQL и подключение к SQL Server или база данных SQL Azure.|  
|**Правка**|Содержит команды для поиска и работа с текстом в страницами с подробными сведениями. Чтобы открыть **управление закладки** диалоговое окно, в меню Правка выберите Управление закладки. В диалоговом окне вы увидите список существующих закладок. Кнопки в правой части диалогового окна можно использовать для управления закладки.|

Также содержит команды для отображения и скрытия **вывода** и **список ошибок** области и в качестве параметра **макеты** для управления с макетами.|  
|**Инструменты**|Содержит команды для создания отчетов, преобразовать схему, обновление из базы данных, миграция объектов и данных и Сохранение скрипта. Также предоставляет доступ к **глобальные параметры, параметры проекта по умолчанию** и **параметры проекта** диалоговым окнам.|  
|**Справка**|Предоставляет доступ к справке SSMA и **о** диалоговое окно.|  
  
### <a name="output-pane-and-error-list-pane"></a>Область вывода и панели «Список ошибок»  
**Представление** предоставляет меню команд для переключения видимости области вывода и области списка ошибок:  
  
-   Область вывода показывает сообщения о состоянии из SSMA во время преобразования объекта синхронизации объектов и переноса данных.  
  
-   В области списка ошибок отображаются ошибки, предупреждения и информационные сообщения в сортируемый список.
