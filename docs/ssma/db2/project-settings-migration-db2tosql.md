# <a name="project-settings-migration-db2tosql"></a>Параметры проекта (миграция) (DB2ToSQL)
Страница миграция диалогового окна **Параметры проекта** содержит параметры, которые позволяют настроить SSMA миграцию данных из DB2 в [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] .  
  
Панель миграция доступна в диалоговых окнах **Параметры проекта** и **Параметры проекта по умолчанию** .  
  
-   Чтобы задать параметры для всех проектов SSMA, в меню **Сервис** выберите **Параметры проекта по умолчанию**, выберите тип проекта миграции, для которого необходимо просмотреть или изменить параметры в раскрывающемся списке **версия целевого объекта миграции** , щелкните **Общие** в нижней части левой панели, а затем щелкните **Миграция**.  
  
-   Чтобы указать параметры для текущего проекта, в меню **Сервис** выберите **Параметры проекта**, щелкните **Общие** в нижней части левой панели, а затем щелкните **Миграция**.  
  
## <a name="migration-engine"></a>Модуль миграции  
  
|Термин|Определение|  
|--------|--------------|  
|**Модуль миграции**|Указывает ядро СУБД, используемое при переносе данных. Перенос данных на стороне клиента относится к клиенту SSMA, получая данные из источника и добавляя их в SQL Server. Миграция данных на стороне сервера относится к подсистеме миграции данных SSMA (программа по копированию), работающей на SQL Server, в качестве задания агента SQL, получая данные из источника и вставляя их непосредственно в SQL Server тем самым избегая дополнительного прыжка клиента (более высокая производительность).<br /><br />**Режим по умолчанию**: модуль миграции данных на стороне клиента<br /><br />**Оптимистичный режим**: модуль миграции данных на стороне клиента<br /><br />**Полный режим**: модуль миграции данных на стороне клиента|  
  
> [!IMPORTANT]  
> Если для параметра **модуль миграции** установлено значение **модуль миграции данных на стороне сервера**, то отображается новый параметр проекта **использовать ядро миграции данных на стороне сервера 32-bit Server** . Он указывает, используется ли служебная программа для переноса данных в 32-разрядной или 64 разрядной программе (BCP).  
  
## <a name="miscellaneous-options"></a>Различные параметры  
  
|Термин|Определение|  
|--------|--------------|  
|**Размер пакета**|Указывает размер пакета, используемый при переносе данных.<br /><br />**Режим по умолчанию**: 10000<br /><br />**Оптимистичный режим**: 10000<br /><br />**Полный режим**: 10000|  
|**Проверочные ограничения**|Указывает, должны ли SSMA проверять ограничения при вставке данных в SQL Server таблицы.<br /><br />**Режим по умолчанию**: false<br /><br />**Оптимистичный режим**: false<br /><br />**Полный режим**: false|  
|**Время ожидания переноса данных**|Указывает время ожидания, используемое при переносе данных<br /><br />**Режим по умолчанию**: 15<br /><br />**Оптимистичный режим**: 15<br /><br />**Полный режим**: 15|  
|**Параметры расширенной миграции данных**|Отображает дополнительные параметры переноса данных для каждой таблицы на отдельной вкладке сведений.<br /><br />**Режим по умолчанию**: скрыть<br /><br />**Оптимистичный режим**: скрытие<br /><br />**Полный режим**: скрыть|  
|**Запускать триггеры**|Указывает, должен ли SSMA запускать триггеры вставки при добавлении данных в SQL Server таблицы.<br /><br />**Режим по умолчанию**: false<br /><br />**Оптимистичный режим**: false<br /><br />**Полный режим**: false|  
|**Сохранять ИД**|Указывает, сохраняет ли SSMA значения NULL в исходных данных при добавлении данных в SQL Server, независимо от значений по умолчанию, указанных в SQL Server.<br /><br />**Режим по умолчанию**: true<br /><br />**Оптимистичный режим**: true<br /><br />**Полный режим**: false|  
|**Сохранять значения NULL**|Указывает, сохраняет ли SSMA значения NULL в исходных данных при добавлении данных в SQL Server, независимо от значений по умолчанию, указанных в SQL Server.<br /><br />**Режим по умолчанию**: true<br /><br />**Оптимистичный режим**: true<br /><br />**Полный режим**: true|  
|**Пометить операцию усечения строк с ошибкой**|Если размер целевого столбца меньше длины исходной строки, значение будет обрезано и отмечено как ошибка.<br /><br />**Режим по умолчанию**: Да<br /><br />**Оптимистичный режим**: Да<br /><br />**Полный режим**: Да|  
|**При ошибке**|Останавливает перенос данных при возникновении ошибки. Он имеет три варианта:<br /><br />**Прерывать миграцию:** Останавливает операцию миграции данных<br /><br />**Перейти к следующей таблице:** Останавливает миграцию данных в текущую таблицу и переходит к следующей<br /><br />**Перейти к следующему пакету:** Останавливает миграцию данных в текущий пакет и переходит к следующему<br /><br />**Режим по умолчанию**: перейти к следующему пакету<br /><br />**Оптимистичный режим**: переход к следующему пакету<br /><br />**Полный режим**: переход к следующему пакету|  
|**Замена неподдерживаемых дат**|Указывает, должны ли SSMA даты исправляться раньше самой ранней [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)] даты **и времени** (01 января 1753).<br /><br />Для сохранения текущих значений дат выберите **не выполнять никаких действий**. [!INCLUDE[ssNoVersion](../../includes/ssnoversion-md.md)]не будет принимать даты до 01 января 1753 в столбце datetime. При использовании старых дат необходимо преобразовать значения DateTime в символьные значения.<br /><br />Чтобы преобразовать даты до 01 января 1753 в NULL, выберите **заменить на null**.<br /><br />Чтобы заменить даты до 01 января 1753 с поддерживаемой датой, выберите вариант **заменить на ближайшую поддерживаемую дату**.<br /><br />**Режим по умолчанию**: не выполнять никаких действий<br /><br />**Оптимистичный режим**: не выполнять никаких действий<br /><br />**Полный режим**: Замените на ближайшую поддерживаемую дату|  
|**Блокировка таблицы**|Указывает, следует ли SSMA блокировать таблицы при добавлении данных в таблицы во время переноса данных. Получает блокировку с массовым обновлением в течение операции с массовым копированием. Если значение равно false, блокировка задается на уровне строк.<br /><br />**Режим по умолчанию**: true<br /><br />**Оптимистичный режим**: true<br /><br />**Полный режим**: true|  
  
## <a name="parallel-data-migration"></a>Параллельная миграция данных  
  
|Термин|Определение|  
|--------|--------------|  
|**Режим параллельной миграции данных**|Указывает режим, используемый для разветвления потоков, чтобы включить параллельную миграцию данных. В режиме auto SSMA выбирает число потоков (по умолчанию — 10), разветвленных для переноса данных. В пользовательском режиме пользователь может указать число потоков, разделенных для переноса данных (минимальное значение — 1, а максимальное — 100). Сейчас только модуль миграции данных на стороне клиента поддерживает параллельную миграцию данных.<br /><br />**Режим по умолчанию**: Авто<br /><br />**Оптимистичный режим**: автоматический<br /><br />**Полный режим**: Авто|  
  
> [!IMPORTANT]  
> Если для параметра **режим параллельной миграции данных** задано значение **Настраиваемый**, то отображается новый параметр проекта параметр **число потоков** . Указывает число потоков, используемых для переноса данных.
