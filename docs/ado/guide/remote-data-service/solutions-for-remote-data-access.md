# <a name="solutions-for-remote-data-access"></a>Решения для удаленного доступа к данным
## <a name="the-issue"></a>Проблема  
 ADO позволяет вашему приложению напрямую получить доступ к и изменение источников данных (иногда называется двухуровневую систему). Например при подключении к источнику данных с вашими данными, который находится в двухуровневую систему прямое подключение.  
  
> [!IMPORTANT]
>  Начиная с Windows 8 и Windows Server 2012, серверные компоненты служб удаленных рабочих СТОЛОВ, больше не включаются в операционной системе Windows (см. в разделе Windows 8 и [настольная книга по совместимости Windows Server 2012](https://www.microsoft.com/download/details.aspx?id=27416) для получения дополнительных сведений). Клиентские компоненты служб удаленных рабочих СТОЛОВ будет поддерживаться в будущих версиях Windows. Избегайте использования этого компонента в новых разработках и запланируйте изменение существующих приложений, в которых он применяется. Приложения, использующие служб удаленных рабочих СТОЛОВ, следует перевести [WCF-сервиса данных](https://go.microsoft.com/fwlink/?LinkId=199565).  
  
 Тем не менее вы можете косвенно обращаться к источникам данных через промежуточное звено, таких как Microsoft® Internet Information Services (IIS). Такой подход иногда называют трехуровневую систему. Наличие IIS является система клиент сервер, который является эффективным способом локального или клиент, приложение может вызывать программу удаленного или сервер, через Интернет или интрасеть. Серверная программа получает доступ к источнику данных и при необходимости обрабатывает данные.  
  
 Например веб-странице интрасети содержит приложения, написанного в Microsoft® Visual Basic Scripting Edition (VBScript), который подключается к службам IIS. IIS в свою очередь подключается к источнику данных, получает данные, обрабатывает их с каким-либо образом и затем возвращает обработанную информацию в приложение.  
  
 В этом примере приложения никогда не непосредственно подключенным к источнику данных; Было сделано IIS. И IIS в доступе к данным с помощью ADO.  
  
> [!NOTE]
>  Клиент серверных приложений не должен быть основан на Интернете или интрасети (то есть веб-) — оно может состоять только из скомпилированного программ в локальной сети. Тем не менее типичный случай — это веб-приложение.  
  
 Так как некоторые визуальному элементу управления, например сетки, "флажок" или список, могут использовать возвращенные сведения, возвращенные сведения необходимо легко использовать по визуальному элементу управления.  
  
 Требуется, чтобы быстро и просто интерфейса программирования приложений, поддерживает три уровня системы, который возвращает сведения о виде легко так, как если бы он был загружен на двухуровневую систему. Службы удаленных данных (RDS) — это интерфейс.  
  
## <a name="the-solution"></a>Решение  
 Служба удаленных рабочих СТОЛОВ определяет модель программирования — последовательность действий, необходимых для получения доступа к и обновления источника данных — для получения доступа к данным через посредник, такие как Internet Information Services (IIS). Модель программирования перечислены все функции платформы RDS.
