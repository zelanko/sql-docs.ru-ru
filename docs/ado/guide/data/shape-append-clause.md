# <a name="shape-append-clause"></a>Предложение APPEND для формирования данных
Предложение APPEND для формирования команды добавляет столбец или столбцы для **записей**. Часто, эти столбцы являются столбцами Глава, указывающие на дочерний элемент **записей**.  
  
## <a name="syntax"></a>Синтаксис  
  
```  
SHAPE [parent-command [[AS] parent-alias]] APPEND column-list  
```  
  
## <a name="description"></a>Описание  
 Ниже приведены компоненты этого предложения.  
  
 *родительский команда*  
 Ноль или один из следующих (можно опустить *родительской команды* полностью):  
  
-   Команды поставщика, заключенный в фигурные скобки (»{}»), возвращающий **записей** объекта. Команды для базового поставщика данных, и его синтаксис зависит от требований этого поставщика. Оно обычно представлено языка SQL, несмотря на то, что ADO не требует любого языка конкретного запроса.  
  
-   Другая команда фигуры, внедренных в круглые скобки.  
  
-   Таблица ключевое слово, за которым следует имя таблицы в поставщике данных.  
  
 *родительский alias*  
 Дополнительный псевдоним, который ссылается на родительский **записей**.  
  
 *Список столбцов*  
 Один или несколько из следующих:  
  
-   Составного столбца.  
  
-   Вычисляемый столбец.  
  
-   Новый столбец, созданные с помощью нового предложения.  
  
-   Столбец. Определение столбца Глава заключено в круглые скобки («()»). См. следующий синтаксис.  
  
```  
SHAPE [parent-command [[AS] parent-alias]]  
   APPEND (child-recordset [ [[AS] child-alias]   
      RELATE parent-column TO child-column | PARAMETER param-number, ... ])  
   [[AS] chapter-alias]   
   [, ... ]  
```  
  
## <a name="remarks"></a>Примечания  
 *дочерних записей*  
 -   Команды поставщика, заключенный в фигурные скобки (»{}»), возвращающий **записей** объекта. Команды для базового поставщика данных, и его синтаксис зависит от требований этого поставщика. Оно обычно представлено языка SQL, несмотря на то, что ADO не требует любого языка конкретного запроса.  
  
-   Другая команда фигуры, внедренных в круглые скобки.  
  
-   Имя существующего форме **записей**.  
  
-   Таблица ключевое слово, за которым следует имя таблицы в поставщике данных.  
  
 *дочерний alias*  
 Псевдоним, относящийся к дочернему объекту **записей**.  
  
 *родительский столбец*  
 Столбец в **записей** возвращаемые *родительской команды.*  
  
 *дочерний столбец*  
 Столбец в **записей** возвращаемые *дочерняя команда*.  
  
 *PARAM-number*  
 См. в разделе [параметризованных команд](../../../ado/guide/data/operation-of-parameterized-commands.md).  
  
 *Глава alias*  
 Псевдоним, который ссылается на столбец добавляется к родительскому элементу.  
  
> [!NOTE]
>  *«Родительский столбец* TO *дочерний столбец»* предложение представляет собой список, где каждый определена связь отделяется запятой  
  
> [!NOTE]
>  Предложение, после ключевого слова APPEND представляет собой список, где каждое предложение разделены запятыми и определяет другой столбец для добавления к родительскому элементу.  
  
## <a name="remarks"></a>Примечания  
 При создании поставщика команд из введенных пользователем данных как часть команды ФИГУРА ФИГУРЫ обрабатывает предоставленное пользователем команды поставщика как непрозрачная строка и точно передавать их в поставщик. Например в следующей команде ФИГУРЫ  
  
```  
SHAPE {select * from t1} APPEND ({select * from t2} RELATE k1 TO k2)  
```  
  
 ФИГУРА будет выполняться две команды: `select * from t1` и (`select * from t2 RELATE k1 TO k2)`. Если пользователь вводит комплексной команду, которая состоит из нескольких команд поставщика, разделенных точкой с запятой, ФИГУРА не является возможность различать разницу. В следующей команде ФИГУРЫ  
  
```  
SHAPE {select * from t1; drop table t1} APPEND ({select * from t2} RELATE k1 TO k2)  
```  
  
 Выполняет ФИГУРЫ `select * from t1; drop table t1` и (`select * from t2 RELATE k1 TO k2),` , не зная `drop table t1` представляет собой отдельную и в этой команде case, опасен, поставщик. Приложения всегда должен проверить пользователя, чтобы предотвратить возникновение таких потенциальных атак злоумышленников.
