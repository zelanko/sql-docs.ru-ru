# <a name="create-partitions"></a>Создание секций

[!INCLUDE[ssas-appliesto-sql2017-later-aas](../../includes/ssas-appliesto-sql2017-later-aas.md)]

На этом занятии создается секций, чтобы разделить таблицу FactInternetSales на более мелкие логические части, которые могут обрабатываться (обновляться) независимо от других секций. По умолчанию каждая таблица, включенная в модель имеет одну секцию, включающую все столбцы и строки таблицы. Для таблицы FactInternetSales нам необходимо разделить данные по годам; одной секции на каждые пять лет таблицы. После этого каждую секцию можно обработать независимо. Дополнительные сведения см. в разделе [Секции](../tabular-models/partitions-ssas-tabular.md). 
  
Предполагаемое время выполнения данного занятия: **15 минут**  
  
## <a name="prerequisites"></a>предварительные требования  

Эта статья входит в учебник по табличному моделированию, который следует изучать в порядке. Прежде чем выполнять задания в этом занятии, необходимо завершить предыдущее занятие: [Занятие 9. Создание иерархий](../tutorial-tabular-1400/as-lesson-9-create-hierarchies.md).  
  
## <a name="create-partitions"></a>Создание секций  
  
#### <a name="to-create-partitions-in-the-factinternetsales-table"></a>Чтобы создать секции в таблице FactInternetSales  
  
1.  В обозревателе табличных моделей разверните **таблиц**, щелкните правой кнопкой мыши **FactInternetSales** > **секций**.  
  
2.  В диспетчере секций щелкните **копирования**, а затем измените имя на **FactInternetSales2010**.
  
    Так как требуется, чтобы секция включала только строки за определенный период, за 2010 год необходимо изменить выражение запроса.
  
4.  Нажмите кнопку **разработки** открыть редактор запросов, а затем нажмите кнопку **FactInternetSales2010** запроса.

5.  В предварительной версии, нажмите кнопку со стрелкой вниз в **OrderDate** заголовок столбца, а затем щелкните **фильтры даты и времени** > **между**.

    ![как lesson10--редактора запросов](../tutorial-tabular-1400/media/as-lesson10-query-editor.png)

6.  В диалоговом окне фильтра строк в **Показать строки, где: OrderDate**, оставьте **позже или одновременно для**и введите в поле даты **1/1/2010**. Оставьте **и** оператор выбран, затем выберите **перед**, введите в поле даты **1/1/2011**и нажмите кнопку **ОК**.

    ![как lesson10-фильтрация строк](../tutorial-tabular-1400/media/as-lesson10-filter-rows.png)
    
    Обратите внимание, что в редакторе запросов, в ПРИМЕНЕННЫХ ДЕЙСТВИЙ, появится еще один шаг, с именем строки с примененным фильтром. Этот фильтр является отобрать только даты заказов за 2010.

8.  нажмите кнопку **Импортировать**;

    В диспетчере секций Обратите внимание, что теперь выражение запроса имеет дополнительное предложение строки с примененным фильтром.

    ![как lesson10-запрос](../tutorial-tabular-1400/media/as-lesson10-query.png)
  
    Эта инструкция указывает, что данная секция должна включать только данные в тех строках, где OrderDate относится к 2010 календарному году, как указано в предложении отфильтрованных строк.  
  
  
#### <a name="to-create-a-partition-for-the-2011-year"></a>Создание секции для 2011 года  
  
1.  В списке секций щелкните **FactInternetSales2010** секции был создан, а затем нажмите кнопку **копирования**.  Сменить имя раздела для **FactInternetSales2011**. 

    Необходимо использовать редактор запросов для создания предложения отфильтрованных строк. Так как вы создали копию запрос для 2010, вам нужно всего лишь внести небольшое изменение в запросе для 2011.
  
2.  В **выражение запроса**, в определенном порядке для этой секции включала только строки для 2011 года, замените значения года в предложении "строки с примененным фильтром" с **2011** и **2012**, соответственно например:  
  
    ```  
    let
        Source = #"SQL/localhost;AdventureWorksDW2014",
        dbo_FactInternetSales = Source{[Schema="dbo",Item="FactInternetSales"]}[Data],
        #"Removed Columns" = Table.RemoveColumns(dbo_FactInternetSales,{"OrderDateKey", "DueDateKey", "ShipDateKey"}),
        #"Filtered Rows" = Table.SelectRows(#"Removed Columns", each [OrderDate] >= #datetime(2011, 1, 1, 0, 0, 0) and [OrderDate] < #datetime(2012, 1, 1, 0, 0, 0))
    in
        #"Filtered Rows"
   
    ```  
  
#### <a name="to-create-partitions-for-2012-2013-and-2014"></a>Для создания секций для 2012, 2013 и 2014.  
  
- Выполните предыдущие шаги, создав секции для 2012, 2013 и 2014 г., изменив значения в предложении "строки с примененным фильтром" для включения только строк за этот год. 
  

## <a name="delete-the-factinternetsales-partition"></a>Удаление секции FactInternetSales

Теперь, когда у вас есть секции для каждого года, можно удалить секцию FactInternetSales; позволяет предотвратить перекрытие при выборе всех процессов, при обработке секций.

#### <a name="to-delete-the-factinternetsales-partition"></a>Для удаления секции FactInternetSales

-  Нажмите кнопку **FactInternetSales** секции, а затем нажмите кнопку **удалить**.



## <a name="process-partitions"></a>Обработка секций  

В диспетчере секций Обратите внимание, что **последняя обработка** столбец для каждой новой секции показывает эти секции никогда не будут обработаны. При создании секций, необходимо запустить операцию обработки секций или обработать таблицу, чтобы обновить данные в этих секциях.  
  
#### <a name="to-process-the-factinternetsales-partitions"></a>Для обработки секций FactInternetSales  
  
1.  Нажмите кнопку **ОК** чтобы закрыть диспетчер секций.  
  
2.  Нажмите кнопку **FactInternetSales** таблицы, а затем щелкните **модели** меню > **процесс** > **обработать секции**.  
  
3.  В диалоговом окне Обработка секций проверьте **режим** присваивается **обработка по умолчанию**.  
  
4.  Установите флажок в столбце **Обработка** для каждой из пяти созданных секций и нажмите кнопку **OK**.  

    ![как lesson10-Обработка секций](../tutorial-tabular-1400/media/as-lesson10-process-partitions.png)
  
    Если будет предложено ввести учетные данные олицетворения, введите имя пользователя Windows и пароль, указанные на занятии 2.  
  
    Откроется диалоговое окно **Обработка данных** со сведениями обработки для каждой секции. Обратите внимание, что для каждой секции передается различное количество строк. Каждая секция включает только строки за год, указанный в предложении WHERE в инструкции SQL. По завершении обработки закройте диалоговое окно "Обработка данных".  
  
    ![как lesson10-процесс — завершение](../tutorial-tabular-1400/media/as-lesson10-process-complete.png)
