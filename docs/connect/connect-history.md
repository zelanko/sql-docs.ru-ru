# <a name="driver-history-for-microsoft-sql-server"></a>Журнал драйверов для Microsoft SQL Server

На этой странице описываются технологии соединений исторических данных Microsoft для подключения к SQL Server.

## <a name="odbc"></a>интерфейс ODBC

Существует три различных поколения драйверы Microsoft ODBC для SQL Server. Первый драйвера ODBC «SQL Server» по-прежнему поставляется как часть [Windows Data Access Components](#microsoft-or-windows-data-access-components). Не рекомендуется использовать именно этот драйвер для разработки новых приложений. Начиная с SQL Server 2005, [собственный клиент SQL Server](#sql-server-native-client) включает интерфейс ODBC и драйвер ODBC, поставляемой вместе с SQL Server 2005 до SQL Server 2012. Не рекомендуется использовать именно этот драйвер для разработки новых приложений. После выпуска SQL Server 2012 [Microsoft ODBC Driver for SQL Server](#microsoft-odbc-driver-for-sql-server) — драйвер, который обновляется с помощью последних компонентов сервера, в дальнейшем.

### <a name="sql-server-native-client"></a>собственный клиент SQL Server

Собственный клиент SQL Server — автономная библиотека, которая используется для OLE DB и ODBC. Собственный клиент SQL Server (часто сокращенное SNAC) был включен в SQL Server 2005, 2012. Собственный клиент SQL Server можно использовать для приложений, требующих, чтобы воспользоваться преимуществами новых функций, появившихся в SQL Server 2005 до SQL Server 2012. (Для этих новых функций в SQL Server не обновляются компоненты доступа к данным Microsoft/Windows.) Для новых возможностей, кроме SQL Server 2012 SQL Server Native Client не обновляется. Переключитесь в Microsoft ODBC Driver для SQL Server или драйвера Microsoft OLE DB для SQL Server, если вы хотите воспользоваться преимуществами новых функций SQL Server, в дальнейшем.

Полную документацию по SQL Server Native Client, см. в разделе [документации по SQL Server Native Client](../relational-databases/native-client/sql-server-native-client-programming.md).

### <a name="microsoft-odbc-driver-for-sql-server"></a>Microsoft ODBC Driver for SQL Server

После выпуска SQL Server 2012 разработали основной драйвер ODBC для SQL Server и выпущена в виде драйвера Microsoft ODBC для SQL Server. Дополнительные сведения см. в разделе [драйвер Microsoft ODBC для документации по SQL Server](./odbc/microsoft-odbc-driver-for-sql-server.md).

## <a name="ole-db"></a>OLE DB

Существует три поколения различных поставщиков Microsoft OLE DB для SQL Server. Первый «Microsoft OLE DB Provider для SQL Server» (SQLOLEDB), по-прежнему поставляется как часть [Windows Data Access Components](#microsoft-or-windows-data-access-components). Этот поставщик не будет обновляться с помощью новых функций и не рекомендуется использовать этот драйвер для разработки новых приложений. Начиная с SQL Server 2005, [собственный клиент SQL Server](#sql-server-native-client) включает интерфейс поставщика OLE DB (SQLNCLI) и поставщика OLE DB, в состав SQL Server 2005 до SQL Server 2017. Это было [, устаревшие в 2011](https://blogs.msdn.microsoft.com/sqlnativeclient/2011/08/29/microsoft-is-aligning-with-odbc-for-native-relational-data-access/) и не рекомендуется использовать этот драйвер для разработки новых приложений. В 2017 г., технология доступа к данным OLE DB впоследствии был [undeprecated и было объявлено о выходе нового выпуска](https://blogs.msdn.microsoft.com/sqlnativeclient/2017/10/06/announcing-the-new-release-of-ole-db-driver-for-sql-server/) 2018 г. Новый поставщик OLE DB называется «Microsoft OLE DB Driver для SQL Server» (MSOLEDBSQL) и находится в настоящее время поддерживаются и поддерживается.

## <a name="adonet"></a>ADO.NET

ADO.NET впервые появилась в Microsoft .NET Framework и продолжает быть улучшены и изменяются. Он является основным компонентом платформы Microsoft .NET Framework. Дополнительные сведения см. в разделе [Microsoft ADO.NET для SQL Server](ado-net/microsoft-ado-net-for-sql-server.md).

## <a name="jdbc"></a>JDBC

### <a name="microsoft-jdbc-driver-for-sql-server"></a>Драйвер Microsoft JDBC для SQL Server

Появились в 2000, Microsoft JDBC Driver для SQL Server продолжает быть улучшены и изменяются. Он был открытым в 2016 г. Последние сведения, включая том, как загрузить драйвер, см. в разделе [Общие сведения о драйвере JDBC](./jdbc/overview-of-the-jdbc-driver.md).

## <a name="php"></a>PHP

### <a name="microsoft-drivers-for-php-for-sql-server"></a>Драйверы Microsoft SQL Server для PHP

Представленный в 2009 г., как это проект с открытым исходным кодом, драйверы Майкрософт для PHP для SQL Server по-прежнему быть улучшены и изменяются. Последние сведения, включая способы скачать драйвер PHP, см. в разделе [драйверы Майкрософт для PHP для SQL Server](./php/microsoft-php-driver-for-sql-server.md).

## <a name="nodejs"></a>Node.js

### <a name="microsoft-driver-for-nodejs-for-sql-server"></a>Драйвер Майкрософт для Node.js для SQL Server

Драйвер Майкрософт для Node.js для SQL Server позволяет приложениям Node.js в Microsoft Windows и Microsoft Windows Azure для доступа к Microsoft SQL Server и Microsoft Windows Azure SQL Database. Процесс разработки ориентированы больше не на этот драйвер. Не рекомендуется создавать новые приложения с помощью драйвера Microsoft для Node.js для SQL Server.

Дополнительные сведения о драйвере Microsoft для Node.js для SQL Server, см. в разделе [WindowsAzure / узел sqlserver](https://github.com/Azure/node-sqlserver).

### <a name="tedious"></a>Утомительно

Корпорация Майкрософт в настоящее время участвует в реализации и поддерживает модуль утомительно открытым исходным кодом в Node.js для подключения к SQL Server с помощью JavaScript. Дополнительные сведения см. в разделе [драйвер Node.js для SQL Server](./node-js/node-js-driver-for-sql-server.md).

## <a name="microsoft-or-windows-data-access-components"></a>Корпорации Майкрософт или компоненты доступа к данным Windows

Microsoft/Windows Data Access Components (MDAC/WDAC) входят в состав и поддерживается в Windows для приложения обратной совместимости и не являются частью текущего стека технологий SQL Server. Компоненты MDAC/WDAC добавляется нет новых функций и не рекомендуется использовать их для разработки новых приложений.

В целях этого документа стек MDAC/WDAC можно разделить на следующие компоненты, на основе технологии и продукты:

* **ADO** (включая ADOMD и ADOX)
* **OLE DB** (включая основными службами OLE DB, SQL Server поставщик OLE DB, поставщик Oracle OLE DB, поставщик OLE DB для драйверов ODBC, поставщик фигур данных и удаленного поставщика данных)
* **ODBC** (включая диспетчер драйверов ODBC, драйвер SQL ODBC и драйвер ODBC для Oracle)

### <a name="mdacwdac-components"></a>Компоненты MDAC/WDAC

Компоненты MDAC/WDAC включает следующие компоненты.

* **ИНТЕРФЕЙС ODBC** Интерфейс Microsoft Open Database Connectivity (ODBC) — это интерфейс языка программирования C, который позволяет приложениям получить доступ к данным из различных систем управления базы данных (СУБД). Приложения, использующие этот API, ограничены только источниками реляционных данных.
* **OLE DB:** OLE DB — это набор COM-интерфейсы для доступа к данным в различных хранилищ данных. Существуют поставщики OLE DB для доступа к данным в базах данных, файловые системы, хранилищ сообщений, службы каталогов, рабочий процесс и хранилищ документов.
* **ADO:** Объекты данных ActiveX (ADO) предоставляет модель программирования высокого уровня. Несмотря на то, что немного меньше высокую производительность, чем писать код в OLE DB или ODBC напрямую, ADO прост в изучении и использовании. Его можно использовать из таких языков скриптов, например Microsoft Visual Basic Scripting Edition (VBScript) или Microsoft JScript.
* **ADOMD:** ADO многомерных данных (ADOMD) будет использоваться с поставщиками многомерных данных, такими как поставщик Microsoft OLAP, также известный как поставщик Microsoft Analysis Services. Нет функциональных усовершенствований момента в него компоненты MDAC 2.0.
* **ADOX:** Расширения ADO для DDL и безопасности (ADOX) позволяют создавать и изменять определения базы данных, таблицы, индекса или хранимой процедуры. ADOX можно использовать с любым поставщиком. Поставщик Microsoft Jet OLE DB обеспечивает полную поддержку ADOX, пока Microsoft SQL Server поставщик OLE DB предоставляет ограниченную поддержку.
* **Сетевые библиотеки Microsoft SQL Server:** Сетевые библиотеки SQL Server позволяют SQLOLEDB и SQLODBC, для взаимодействия с базой данных SQL Server. Рекомендуется использовать следующие сетевые библиотеки SQL Server в версиях MDAC/WDAC: Banyan Vines, AppleTalk, ServerNet, IPX/SPX, Giganet и RPC. TCP/IP и именованные каналы будут по-прежнему поддерживаются и доступны в 64-разрядной операционной системе Windows.
* **MSDASQL:** Поставщик Microsoft OLE DB для ODBC (MSDASQL) позволяет приложениям, созданным в OLE DB и ADO (который внутренне используется OLEDB) обращаться к источникам данных посредством драйвера ODBC. MSDASQL является поставщиком OLEDB, подключающийся к ODBC, вместо базы данных. Он предназначен как мост из OLE DB для драйвера ODBC при существует прямой поставщик OLE DB для источника данных. MSDASQL поставляется с операционной системой Windows и Windows Server 2008 и Vista с пакетом обновления 1 были освобождает первый Windows для включения 64-разрядной версии технологии.

### <a name="deprecated-mdacwdac-components"></a>Устаревшие компоненты MDAC/WDAC

Эти компоненты по-прежнему поддерживаются в текущем выпуске MDAC/WDAC, но они может быть удален в будущих выпусках. При разработке новых приложений, корпорация Майкрософт рекомендует избегать использования этих компонентов. Кроме того при обновлении или изменению существующих приложений, исключите все зависимости от этих компонентов.

* **SQLOLEDB:** Поставщик Microsoft OLE DB для SQL Server (SQLOLEDB), которая поддерживает доступ к Microsoft SQL Server, является устаревшим. Возможно, его подключении к будущим версиям SQL Server не поддерживается. Возможность подключения к версии, более ранних, чем SQL Server 7 будут удалены из операционной системы после Windows 7. Новые приложения должны использовать драйвер Microsoft OLE DB для SQL Server (MSOLEDBSQL), которая поддерживает новые функции SQL Server. Существующие приложения, следует перевести в драйвере Microsoft OLE DB для SQL Server также для повышения производительности, надежности и возможности поддержки. Дополнительные сведения см. в разделе [обновление приложения для драйвера OLE DB для SQL Server с компонентами MDAC](oledb/applications/updating-an-application-to-oledb-driver-for-sql-server-from-mdac.md).
* **SQLODBC:** Microsoft SQL Server ODBC Driver (SQLODBC), которая поддерживает доступ к Microsoft SQL Server, является устаревшим. Возможно, его подключении к будущим версиям SQL Server не поддерживается. Возможность подключения к версии, более ранних, чем SQL Server 7 будут удалены из операционной системы после Windows 7. Новые приложения должны использовать Microsoft ODBC Driver for SQL Server в Windows, которая поддерживает новые функции SQL Server. Существующие приложения следует перевести драйвера Microsoft ODBC для SQL Server также для повышения производительности, надежности и возможности поддержки. Важные сведения, см. в разделе [обновление приложения собственного клиента SQL Server с компонентами MDAC](../relational-databases/native-client/applications/updating-an-application-to-sql-server-native-client-from-mdac.md).
* **Базы данных Microsoft Jet 4.0:** Начиная с версии 2.6, не содержат компоненты Jet. Другими словами компоненты MDAC 2.6 и 2.7, 2.8 не содержат Microsoft Jet, поставщик Microsoft Jet OLE DB, драйверы для баз данных ODBC рабочего стола или Jet объектов доступа к данным (DAO). 

  Доступно не 64-разрядной версии базы данных Jet, драйвер OLEDB для Jet, драйверы ODBC Jet или Jet DAO. Дополнительные сведения см. в [статье базы знаний 957570](https://support.microsoft.com/kb/957570). На 64-разрядных версиях Windows выполняется Jet 32-разрядной подсистеме WOW64 на Windows. Дополнительные сведения о WOW64 см. в разделе [документации MSDN WOW64](/windows/desktop/WinProg64/wow64-implementation-details). Собственный 64-разрядные приложения не могут взаимодействовать с 32-разрядные драйверы Jet, работающих в эмуляторе WOW64.

  Вместо Microsoft Jet, корпорация Майкрософт рекомендует использовать [Microsoft SQL Server Express Edition](https://www.microsoft.com/sql-server/sql-server-editions-express) при разработке новых приложений Microsoft Access, которым требуется реляционное хранилище данных. Эти новые или преобразованного приложения Jet можно продолжать использовать Jet целью с помощью Microsoft Office 2003 и более ранних версий файлов (.mdb и XLS) для хранения данных неосновной. Тем не менее для этих приложений, необходимо перенести из Jet в системный драйвер Office 2007. Вы можете [загрузить системный драйвер Office 2007](https://www.microsoft.com/downloads/details.aspx?displaylang=en&FamilyID=7554f536-8c28-4598-9b72-ef94e038c891), позволяющий выполнять чтение и запись в существующие файлы в Office 2003 (.mdb и XLS) или форматов файлов Office 2007 (*.accdb, *.xlsm, *.xlsx и *.xlsb).

  > [!IMPORTANT]
  > Прочитайте 2007 Office System лицензионное соглашение для ограничения определенных использования.

  > [!NOTE]
  > Приложения SQL Server также можно получить доступ к системе Office 2007 и более ранних версиях файлы из подключения к SQL Server разнородных данных и служб интеграции возможностях, а также через системный драйвер Office 2007. Кроме того 64-разрядных приложений SQL Server можно обращаться для 32-разрядных Jet и файлов выпуска 2007 системы Office с помощью 32-разрядных SQL Server Integration Services (SSIS) на 64-разрядной Windows.

* **MSDADS:** С помощью поставщика Microsoft OLE DB для данных формирование (MSDADS) можно создать иерархические связи между ключи, поля или наборы строк в приложении. Нет функциональных усовершенствований были сделаны после MDAC 2.1. Этот поставщик является устаревшим. Корпорация Майкрософт рекомендует использовать XML, вместо MSDADS.
* **Oracle OLE DB, ODBC и Oracle:** Драйвер Microsoft ODBC Oracle (Oracle ODBC) и поставщик Microsoft OLE DB для Oracle (Oracle OLE DB) предоставляют доступ к серверам баз данных Oracle. Они создаются с помощью вызова интерфейс Oracle (OCI) версии 7 и обеспечивать полную поддержку Oracle 7. Кроме того он использует эмуляции Oracle 7 для обеспечивают ограниченную поддержку для баз данных Oracle 8. Oracle не поддерживает приложения, использующие вызовы OCI версии 7. Эти технологии являются устаревшими. Если вы используете источники данных Oracle, необходимо перенести драйвера Oracle и поставщик.
* **RDS:** Remote Data Services (RDS) — это собственный механизм Microsoft, для доступа к удаленной объекты ADO Recordset через Интернет или интрасеть. Служба удаленных рабочих СТОЛОВ устарел. нет функциональных усовершенствований момента для служб удаленных рабочих СТОЛОВ MDAC 2.1. Корпорация Майкрософт выпустила .NET Framework, которая имеет расширенные возможности SOAP и заменяет компоненты служб удаленных рабочих СТОЛОВ. Все серверные компоненты служб удаленных рабочих СТОЛОВ будут удалены из операционной системы после Windows 7.
* **JRO:** Объекты репликации Jet (JRO) является устаревшим. В ADO с Jet используется JRO (*.mdb) баз данных для создания и сжатия базы данных Jet (.mdb) и выполнять управление репликацией Jet. MDAC 2.7 будет последнего выпуска. JRO будет недоступен в 64-разрядной операционной системе Windows. JRO не поддерживается в формате Microsoft Access 2007 (*.accdb).
* **Поддержка 16-разрядная версия ODBC:** Если вы используете 16-разрядных приложений, следует мигрировать на 32-разрядное приложение. 16-разрядных функций является устаревшим и удаляется из 64-разрядных операционных системах. Дополнительные сведения см. в [статье базы знаний 896458](https://support.microsoft.com/kb/896458).
* **Простой поставщик OLEDB (MSDAOSP):** Простой поставщик OLE DB представляет собой платформу для быстрого создания поставщиков OLE DB через простых данных. MSDAOSP является устаревшим.
* **Библиотека курсоров ODBC** Библиотека курсоров ODBC (ODBCCR32.dll) предоставляет курсоры только данные клиента. Библиотека курсоров ODBC оно устарело; приложение может использовать способа реализации курсоров на стороне сервера для замены.
* **OLE DB интерфейса вне процесса удаленного взаимодействия:** OLEDB интерфейс удаленного взаимодействия (msdaps.dll) была предпринята попытка разрешить поставщики OLE DB для выполнения вне процесса. Удаленное взаимодействие OLEDB интерфейс вне процесса является устаревшим.
* **AppleTalk и Banyan Vines сетевых библиотек SQL:** Сетевые библиотеки Banyan Vines, AppleTalk, ServerNet, IPX/SPX, Giganet и RPC SQL являются устаревшими. При использовании любой из этих технологий, следует изменять приложения могут использовать один из других сетевые библиотеки, такие как TCP/IP и именованных каналов.

### <a name="mdacwdac-releases"></a>Выпуски MDAC/WDAC

Ниже приведен список сценариев поддержки последних выпусков MDAC/WDAC, начиная с самой ранней.

* **MDAC 1.5, компоненты MDAC 2.0 и MDAC 2.1:** Эти версии MDAC были независимых выпусков, которые были выпущены в Microsoft Windows NT Option Pack, пакет SDK для платформы Microsoft Windows или MDAC веб-сайта. Эти версии MDAC, больше не поддерживаются.
* **MDAC 2.5:** Эта версия MDAC вошло в состав операционной системы Windows 2000. Пакеты обновления для MDAC 2.5 входили в состав пакетов обновления Windows 2000.
* **Компоненты MDAC 2.6:** MDAC 2.6 RTM, SP1 и SP2 входили в состав Microsoft SQL Server 2000 RTM, SP1 и SP2, соответственно. Кроме того эти пакеты обновлений MDAC были выпущены для MDAC веб-сайта соответствии с расписанием выпуска пакета обновления Microsoft SQL Server 2000. Эта версия MDAC и пакеты обновления можно установить на платформах Windows 2000, Windows Millennium Edition, Windows NT, Windows 95 и Windows 98. Эта версия MDAC больше не поддерживается.
* **MDAC 2.7:** Эта версия MDAC вошло в состав операционных систем Microsoft Windows XP RTM и SP1. Эта версия MDAC и пакеты обновления можно установить на платформах Windows 2000, Windows Millennium, Windows NT и Windows 98. Эту версию можно установить на платформе Windows XP только с помощью операционной системы или пакетами служб. Эта версия MDAC больше не поддерживается.
* **MDAC 2.8:** Эта версия MDAC был вместе с Windows Server 2003 и Windows XP SP2 и более поздних версий. Эта версия MDAC и пакеты обновления также можно установить на Windows 2000.

  * 32-разрядной версии MDAC 2.8 также была выпущена на MDAC веб-сайт в то же время, Windows Server 2003 был выпущен для клиента.
  * 64-разрядной версии MDAC 2.8 был выпущен вместе с 64-разрядной версии Windows Server 2003 и Windows XP.

* **Компоненты доступа к данным Windows DAC** Компоненты MDAC изменил его имя, чтобы WDAC - «Windows компоненты доступа к данным» начиная с Windows Vista и Windows Server 2008. WDAC включено как часть операционной системы и не поддерживается для распространения по отдельности. Удобство в обслуживании WDAC подчиняется жизненного цикла операционная система.

  32-разрядных и 64-разрядных версиях WDAC выпускаются с 32-разрядных и 64-разрядных версиях операционных систем Windows, соответственно.

## <a name="obsolete-data-access-technologies"></a>Технологии доступа к устаревшим данным.

Устаревшие технологии — технологии, не были улучшены или обновлены в некоторые выпуски продуктов, и, будут исключены из будущие версии продукта. Не использовать эти технологии, при создании новых приложений. После внесения изменений в существующие приложения, написанные с использованием этих технологий, рассмотрите миграцию приложений ADO.NET или другой текущей технологией.

Следующие компоненты считаются устаревшими.

* **DB-Library:** DB-Library — это модель программирования связанные с SQL Server, который включает API C. Начиная с версии SQL Server 6.5 были не функциональные улучшения для DB-Library. Предшествующее выпуску окончательной версии была с SQL Server 2000, и он не будет переноситься в 64-разрядной операционной системы Windows.
* **Embedded SQL (E-SQL):** E-SQL — это модель программирования связанные с SQL Server, которая позволяет инструкций Transact-SQL, внедренных в код Visual C. Нет функциональные улучшения были сделаны в E-SQL после SQL Server 6.5. Предшествующее выпуску окончательной версии была с SQL Server 2000, и он не будет переноситься в 64-разрядной операционной системы Windows.
* **Объекты доступа к данным (DAO):** DAO предоставляет доступ к базам данных JET (доступ). Этот API можно использовать из Microsoft Visual Basic, Microsoft Visual C++ и языки сценариев. Он был включен с помощью Microsoft Office 2000 и Office XP. DAO 3.6 является окончательной версии этой технологии. Он будет недоступен в 64-разрядной операционной системе Windows.
* **Объекты удаленных данных (RDO):** RDO был разработан специально для доступа к удаленной ODBC реляционные источники данных и стала проще с помощью ODBC без кода сложного приложения. Он вошло в состав Microsoft Visual Basic версии 4, 5 и 6. RDO версии 2.0 была окончательная версия этой технологии.
