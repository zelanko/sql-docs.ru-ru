# <a name="criteria-pane-visual-database-tools"></a>Панель критериев (визуальные инструменты для баз данных)
[!INCLUDE[appliesto-ss-asdb-asdw-pdw-md](../../includes/appliesto-ss-asdb-asdw-pdw-md.md)]
Вводя значения в табличную сетку, на панели критериев можно указывать параметры запроса, например: какие из столбцов данных следует отображать, как упорядочивать результаты, какие строки выделить. На панели критериев можно указать:  
  
-   отображаемые столбцы и псевдонимы имен столбцов;  
  
-   таблицу, которой принадлежит столбец;  
  
-   выражения для вычисляемых столбцов;  
  
-   порядок сортировки для запроса;  
  
-   условия поиска;  
  
-   критерии группирования, включая агрегатные функции для сводных отчетов;  
  
-   новые значения для запросов UPDATE или INSERT INTO;  
  
-   имена целевых столбцов для запросов INSERT FROM.  
  
Изменения, произведенные на панели критериев, автоматически отображаются на панели диаграмм и панели SQL. Таким же образом панель критериев автоматически обновляется, отображая изменения, произведенные на других панелях.  
  
## <a name="about-the-criteria-pane"></a>О панели критериев  
Строки на панели критериев отображают столбцы данных, используемые в запросе: в строках панели критериев отображаются параметры запросов.  
  
Специальные дополнительные сведения, отображаемые на панели критериев, зависят от типа создаваемого запроса.  
  
Если панель критериев не видна, щелкните конструктор правой кнопкой мыши, укажите **Панель**и выберите пункт **Критерии**.  
  
## <a name="options"></a>Параметры  
  
|**Столбец**|**Тип запроса**|**Описание**|  
|--------------|------------------|-------------------|  
|Столбец|All|Отображает либо имя столбца данных, используемого для запроса, либо выражение для вычисляемого столбца. Этот столбец зафиксирован и всегда виден независимо от горизонтальной прокрутки.|  
|Псевдоним|SELECT, INSERT FROM, UPDATE, MAKE TABLE|Задает альтернативное имя либо для столбца, либо для вычисляемого столбца.|  
|Таблица|SELECT, INSERT FROM, UPDATE, MAKE TABLE|Задает имя таблицы или табличного объекта для связанного столбца данных. Для вычисляемых столбцов этот столбец пуст.|  
|Выходные данные|SELECT, INSERT FROM, MAKE TABLE|Определяет, добавляется ли столбец данных в результат запроса.<br /><br />Если это допускается базой данных, столбец данных может использоваться в выражениях сортировки или поиска без его отображения в результирующем наборе.|  
|Тип сортировки|SELECT, INSERT FROM|Определяет, используется ли связанный столбец данных при сортировке результатов запроса, а также порядок сортировки: по возрастанию или убыванию.|  
|Порядок сортировки|SELECT, INSERT FROM|Определяет приоритеты столбцов данных, используемых при сортировке результирующего набора. При изменении порядка сортировки для столбца данных порядок сортировки для всех остальных столбцов соответствующим образом обновляется.|  
|Группирование|SELECT, INSERT FROM, MAKE TABLE|Задает использование соответствующего столбца данных для создания статистического запроса. Этот столбец отображается в сетке только в том случае, если выбран параметр **Group By** в меню **Инструменты** или на панели SQL было добавлено предложение GROUP BY.<br /><br />По умолчанию этот столбец устанавливается в значение **Group By**и становится частью предложения GROUP BY.<br /><br />При перемещении к ячейке в этом столбце и выборе агрегатной функции, применяемой к соответствующему столбцу данных, по умолчанию результирующее выражение добавляется к результирующему набору как выходной столбец.|  
|Критерии|All|Задает условия поиска (фильтр) для соответствующего столбца данных. Добавляет оператор (по умолчанию «=») и значение для поиска. Текстовые значения необходимо заключать в одинарные кавычки.<br /><br />Если соответствующий столбец данных входит в предложение GROUP BY, введенное выражение используется в инструкции HAVING.<br /><br />Если в сетке введены значения для нескольких ячеек в столбце **Критерии**, условия поиска автоматически связываются логической операцией И.<br /><br />Чтобы задать несколько условий поиска для одного столбца базы данных, например (fname > 'A') AND (fname < 'M'), нужно дважды добавить столбец на панель критериев и для каждого вхождения столбца данных ввести разные значения в столбце **Критерии**.|  
|Или...|All|Определяет дополнительные условия поиска для столбца данных, связанные с предыдущими условиями логической операцией ИЛИ. В сетку можно добавить несколько столбцов **Или...** , нажав клавишу TAB, находясь в самом правом столбце **Или...**|  
|Append|INSERT FROM|Задает имя целевого столбца данных для соответствующего столбца данных. При создании запроса типа INSERT FROM конструктор запросов и представлений пытается сопоставить источник соответствующему целевому столбцу данных. Если конструктор запросов и представлений не может выбрать соответствие, необходимо указать имя столбца.|  
|Новое значение|UPDATE, INSERT INTO|Определяет значение, вставляемое в соответствующий столбец. Введите значение литерала или выражение.|
