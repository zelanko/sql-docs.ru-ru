# <a name="aggregate-values-in-a-dataset-by-using-the-aggregate-transformation"></a>Статистическая обработка значений в наборе данных с помощью преобразования «Агрегатная обработка»
  Для добавления и настройки преобразования «Агрегатная обработка» пакет должен обладать как минимум одной задачей потока данных и одним источником.  
  
### <a name="to-aggregate-values-in-a-dataset"></a>Выполнение статистического вычисления значений в наборе данных  
  
1.  В среде [!INCLUDE[ssBIDevStudioFull](../../../includes/ssbidevstudiofull-md.md)]откройте проект служб [!INCLUDE[ssISnoversion](../../../includes/ssisnoversion-md.md)] , содержащий необходимый пакет.  
  
2.  Чтобы открыть пакет, дважды щелкните его в обозревателе решений.  
  
3.  Перейдите на вкладку **Поток данных** и перетащите преобразование «Статистическая обработка» из **области элементов**в область конструктора.  
  
4.  Подключите преобразование «Агрегатная обработка» к потоку данных, перетащив соединитель из источника или предыдущего преобразования в преобразование «Агрегатная обработка».  
  
5.  Дважды щелкните преобразование.  
  
6.  В диалоговом окне **Редактор преобразования «Статистическая обработка»** перейдите на вкладку **Статистические выражения** .  
  
7.  В списке **Доступные входные столбцы** установите флажки рядом со столбцами, для которых нужно выполнить статистическую обработку значений. Отмеченные столбцы появятся в таблице.  
  
    > [!NOTE]  
    >  Можно отметить столбец несколько раз, применяя, таким образом, к нему множественные преобразования. Для уникального определения статистических выражений к данному по умолчанию названию для псевдонима выхода столбца добавляется число.  
  
8.  При необходимости измените значение в столбцах **Псевдоним выхода** .  
  
9. Для изменения установленной по умолчанию статистической операции **Group by**выберите другую операцию в списке **Операция** .  
  
10. Для изменения сравнения по умолчанию выберите отдельные флаги сравнения, перечисленные в столбце **Флаги сравнения** . По умолчанию сравнение не учитывает регистр, тип японской азбуки, несамостоятельные знаки и ширину символов.  
  
11. При необходимости определите для статистического выражения **Подсчет различных объектов** точное число различающихся значений в столбце **Количество различных ключей** или выберите приблизительное число в столбце **Масштаб различных ключей** .  
  
    > [!NOTE]  
    >  Указание точного или приблизительного числа различных значений оптимизирует производительность, поскольку преобразование может заранее выделить память, необходимую для выполнения задачи.  
  
12. При необходимости щелкните мышью **Дополнительно** и обновите название выхода преобразования «Статистическая обработка». Если агрегаты включают `Group By` операцию, можно выбрать приблизительное число значений ключа группирования в столбце **Масштаб ключей** или указать точное число ключевых значений группирования в столбце **ключи** .  
  
    > [!NOTE]  
    >  Указание точного или приблизительного числа различных значений оптимизирует производительность, поскольку преобразование может заранее выделить память, необходимую для выполнения задачи.  
  
    > [!NOTE]  
    >  Параметры **Порядок числа ключей** и **Ключи** взаимоисключающие. Если значения заданы как в столбце **Порядок числа ключей** , так и в столбце **Ключи** , используется большее значение.  
  
13. При необходимости щелкните мышью вкладку **Дополнительно** и установите атрибуты, которые относятся к оптимизации всех операций преобразования «Статистическая обработка».  
  
14. Нажмите кнопку **ОК**.  
  
15. Чтобы сохранить обновленный пакет, выберите пункт **Сохранить выбранные элементы** в меню **Файл** .
