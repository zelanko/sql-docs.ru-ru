# <a name="create-table---sql-command"></a>CREATE TABLE (команда SQL)
Создает таблицу с указанными полями.  
  
 Драйвер ODBC для Visual FoxPro поддерживает собственный синтаксис языка Visual FoxPro для этой команды. Сведения, см. в разделе **"Примечания" драйвер**.  
  
## <a name="syntax"></a>Синтаксис  
  
```  
  
CREATE TABLE | DBF TableName1 [NAME LongTableName] [FREE]  
   (FieldName1FieldType [(nFieldWidth [, nPrecision])]  
      [NULL | NOT NULL]   
      [CHECK lExpression1 [ERROR cMessageText1]]  
      [DEFAULT eExpression1]  
      [PRIMARY KEY | UNIQUE]  
      [REFERENCES TableName2 [TAG TagName1]]  
      [NOCPTRANS]  
   [, FieldName2 ...]  
      [, PRIMARY KEY eExpression2 TAG TagName2  
      |, UNIQUE eExpression3 TAG TagName3]  
      [, FOREIGN KEY eExpression4 TAG TagName4 [NODUP]  
            REFERENCES TableName3 [TAG TagName5]]  
      [, CHECK lExpression2 [ERROR cMessageText2]])  
| FROM ARRAY ArrayName  
```  
  
## <a name="arguments"></a>Аргументы  
 Создание таблицы &#124; DBF *TableName1*  
 Задает имя создаваемой таблицы. Параметры таблицы и DBF идентичны.  
  
 ИМЯ *LongTableName*  
 Указывает длинное имя для таблицы. Можно указать имя большой таблицы только в том случае, если база данных открыта, так как имена большой таблицы хранятся в базах данных.  
  
 Длинные имена могут содержать до 128 символов и может использоваться вместо короткие имена файлов в базе данных.  
  
 FREE  
 Указывает, что таблицы не будут добавляться в открытую базу данных. БЕСПЛАТНЫЙ не требуется, если базы данных еще не открыт.  
  
 *(FieldName1 FieldType* [( *nFieldWidth* [, *nPrecision*])]  
 Указывает имя поля, тип поля, ширины поля и поля, точность (количество знаков после запятой), соответственно.  
  
 *FieldType* представляет одну букву, указывающее поля [тип данных](../../odbc/microsoft/visual-foxpro-field-data-types.md). Некоторые типы данных полей необходимо указать *nFieldWidth* или *nPrecision* или оба.  
  
 *nFieldWidth* и *nPrecision* для D, G, я, L, M, P, T и Y игнорируются типов. *nPrecision* по умолчанию (без дробных разрядов), если *nPrecision* не будет включено для типов B "," F "или" N.  
  
 NULL  
 Допускает значения null в поле.  
  
 NOT NULL  
 Предотвращает значения null в поле.  
  
 Если опустить NULL и NOT NULL, текущее значение параметра SET NULL определяет, допустимы ли значения null в поле. Тем не менее, если опустить NULL и NOT NULL и содержит первичный ключ или УНИКАЛЬНЫЙ предложение, текущее значение параметра SET NULL учитывается и поле по умолчанию NOT NULL.  
  
 ПРОВЕРЬТЕ *lExpression1*  
 Указывает правило проверки для поля. *lExpression1* может быть определяемой пользователем функции. Каждый раз, когда добавляется пустую запись, проверяются правила проверки. Если правило проверки не допускает наличии пустых значений в записи о присоединенных, формируется ошибка.  
  
 Ошибка *cMessageText1*  
 Указывает сообщение об ошибке отображается Visual FoxPro, когда правило поля приводит к ошибке. Сообщение отображается только в том случае, когда данные изменяются в окне «Обзор» или окне редактирования.  
  
 По умолчанию *eExpression1*  
 Задает значение по умолчанию для поля. Тип данных *eExpression1* должен совпадать с типом данных поля.  
  
 PRIMARY KEY  
 Создает первичный индекс для поля. Тег первичный индекс имеет имя, что и поле.  
  
 UNIQUE  
 Создает индекс кандидатом для поля. Тег индекс кандидата имеет тем же именем, что и поле.  
  
> [!NOTE]  
>  Кандидат индексов (создать, добавив УНИКАЛЬНЫЕ возможности в CREATE TABLE или ALTER TABLE - SQL) не так же, как индексы, созданные с параметром, УНИКАЛЬНЫЙ в команде ИНДЕКСА. Индекс создан с параметром, УНИКАЛЬНЫЙ в команде индекс допускает повторяющиеся ключи индекса; повторяющиеся ключи индекса не позволяют возможных кандидатур индексов. См. в разделе [индекс](../../odbc/microsoft/index-command.md) Дополнительные сведения о его УНИКАЛЬНЫЙ параметр.  
  
 В поле, используемое для индекса первичные или потенциальные не разрешены значения NULL и повторяющиеся записи. Тем не менее Visual FoxPro не будет формировать ошибку, если при создании индекса первичные или потенциальные для поля, поддерживающего значения null. Visual FoxPro приведет к ошибке при попытке ввести значение null или повторяющиеся в поля, используемого для первичные или потенциальные индекса.  
  
 Ссылки на *TableName2*[ТЕГ *TagName1*]  
 Указывает родительской таблицы, к которому устанавливается постоянных связь. Если опустить ТЕГ *TagName1*, связь устанавливается с помощью первичный ключ родительской таблицы. Если родительская таблица не имеет первичного индекса, Visual FoxPro приводит к ошибке.  
  
 Следует включить ТЕГ *TagName1* для установления отношения, в зависимости от существующей метки индекса для родительской таблицы. Имена тегов индекс может содержать до 10 символов.  
  
 Родительской таблицы не может быть произвольным.  
  
 NOCPTRANS  
 Запрещает преобразование в другую кодовую страницу для символов и типа memo полей. Если таблицы преобразуется в другой кодовой страницы, поля, для которых был указан NOCPTRANS не переводятся. NOCPTRANS можно указать только для полей символов и типа memo.  
  
 Следующий пример создает таблицу с именем mytable, содержащий два поля символ, а также два поля типа memo. Во втором поле символа, char2 и второе поле типа memo memo2, включают NOCPTRANS для предотвращения перевода.  
  
```  
CREATE TABLE mytable (char1 C(10), char2 C(10) NOCPTRANS,;  
   memo1 M, memo2 M NOCPTRANS)  
```  
  
 ПЕРВИЧНЫЙ ключ *eExpression2* ТЕГА *TagName2*  
 Задает первичный индекс для создания. *eExpression2* указывает любому полю или сочетание полей в таблице. ТЕГ *TagName2* указывает имя для тега первичного индекса, который создается. Имена тегов индекс может содержать до 10 символов.  
  
 Поскольку таблица может иметь только один первичный индекс, нельзя включать это предложение, если вы уже создали первичный индекс для поля. Visual FoxPro приводит к ошибке, если содержат более одного предложения ПЕРВИЧНОГО ключа в инструкции CREATE TABLE.  
  
 УНИКАЛЬНЫЙ *eExpression3*ТЕГА *TagName3*  
 Создает индекс кандидатов. *eExpression3* указывает любому полю или сочетание полей в таблице. Тем не менее если вы создали первичный индекс с одним из параметров PRIMARY KEY, не может включать поле, которое было указано в индекс первичного. ТЕГ *TagName3* задает имя тега для тега кандидата индекс, созданный. Имена тегов индекс может содержать до 10 символов.  
  
 Таблица может иметь несколько возможных кандидатур индексов.  
  
 ВНЕШНИЙ ключ *eExpression4*ТЕГА *TagName4*[NODUP]  
 Создает внешний индекс (непервичных) и устанавливает связь с родительской таблицей. *eExpression4* указывает внешний индекс ключевое выражение и *TagName4* указывает имя ключа тега внешнего индекса, который создается *.* Имена тегов индекс может содержать до 10 символов. Включить NODUP для создания внешнего индекса кандидатов.  
  
 Можно создать несколько внешних индексов для таблицы, но выражения внешнего индекса необходимо указать разные поля в таблице.  
  
 Ссылки на *TableName3*[ТЕГ *TagName5*]  
 Указывает родительской таблицы, к которому устанавливается постоянных связь. Следует включить ТЕГ *TagName5* для установления отношения зависимости с тегом индекса для родительской таблицы. Имена тегов индекс может содержать до 10 символов. По умолчанию, если не указан ТЕГ *TagName5,* установления связи с помощью индекс первичного ключа в родительской таблице.  
  
 ПРОВЕРЬТЕ *eExpression2*[ошибка *cMessageText2*]  
 Указывает правило проверки таблицы. Ошибка *cMessageText2* указывает сообщение об ошибке, которое отображает Visual FoxPro, когда выполняется правило проверки таблицы. Сообщение отображается только в том случае, когда данных изменяется в окне «Обзор» или окна редактирования.  
  
 ИЗ МАССИВА *ArrayName*  
 Указывает имя существующего массива, содержимое которой представлено имя, тип, точность и масштаб для каждого поля в таблице. Содержимое массива может быть определена с **AFIELDS**()-функция.  
  
## <a name="remarks"></a>Примечания  
 Новая таблица открывается в минимально доступные рабочие области и может осуществляться с помощью псевдонима. Новая таблица открывается в монопольном режиме, независимо от текущего значения УСТАНОВИТЬ МОНОПОЛЬНУЮ.  
  
 Если база данных открыта и не включают бесплатные предложения, новая таблица добавляется в базу данных. Не удается создать таблицу с тем же именем, как таблицы в базе данных.  
  
 Если база данных открыта, CREATE TABLE - SQL требует монопольного использования базы данных. Чтобы открыть базу данных для монопольного использования, включающий ИСКЛЮЧАЮЩИЙ в базе данных, ОТКРОЙТЕ.  
  
 Если базы данных не открыт, при создании новой таблицы, включая имя, проверки, по умолчанию, FOREIGN KEY, PRIMARY KEY или ссылок на предложения приводит к ошибке.  
  
> [!NOTE]  
>  Синтаксис инструкции CREATE TABLE используются запятые для разделения параметров определенных CREATE TABLE. Кроме того значение NULL, не NULL, проверка, по умолчанию, первичный ключ и УНИКАЛЬНЫЕ предложения должен быть помещен в круглые скобки, содержащая определения столбцов.  
  
## <a name="driver-remarks"></a>Драйвер "Примечания"  
 Когда приложение отправляет инструкции ODBC SQL CREATE TABLE в источник данных, драйвер ODBC для Visual FoxPro преобразует команды в ТАБЛИЦЕ Visual FoxProCREATE команду, используя синтаксис, показанный в следующей таблице.  
  
|Синтаксис ODBC|Синтаксис Visual FoxPro|  
|-----------------|--------------------------|  
|CREATE TABLE *base-table-name*<br /><br /> (*идентификатор столбца типа данных*<br /><br /> [NOT NULL]<br /><br /> [,*идентификатор столбца типа данных*<br /><br /> [NOT NULL]...)|Создание таблицы *TableName1* [имя *LongTableName*]<br /><br /> (*FieldName1* *FieldType*<br /><br /> [(*nFieldWidth* [, *nPrecision*])]<br /><br /> [НЕ NULL)]|  
  
 При создании таблицы с помощью драйвера, драйвер закроет таблицу сразу после создания, чтобы разрешить доступ к ней другие пользователи. Это отличается от Visual FoxPro, который оставляет исключительно при создании таблицы. Тем не менее если выполняется хранимая процедура для источника данных, содержащий инструкцию CREATE TABLE, таблице остается открытым.  
  
 Если источник данных является базой данных (файл .dbc), драйвер ODBC для Visual FoxPro создает таблицу с именем *LongTableName* с тем же именем, что *base-table-name*.  
  
### <a name="using-data-definition-language-ddl"></a>С помощью языка описания данных (DDL)  
 DDL не могут содержать в следующих местах:  
  
-   В пакете инструкцию SQL, требуется транзакция  
  
-   В режиме ручной фиксации, после оператора, который требуется транзакция, в том случае, если приложение сначала вызывает **SQLTransact**.  
  
 Например если вы хотите создать временную таблицу, следует создать таблицу, перед началом транзакции, для которой требуются. Если включить инструкцию CREATE TABLE в пакете SQL, который требуется транзакция, драйвер возвращает сообщение об ошибке.
