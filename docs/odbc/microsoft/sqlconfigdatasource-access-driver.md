# <a name="sqlconfigdatasource-access-driver"></a>SQLConfigDataSource (драйвер для Access)
> [!NOTE]  
>  В этом разделе сведения конкретного драйвера доступа. Общие сведения об этой функции см. в соответствующем разделе [Справочник по API ODBC](../../odbc/reference/syntax/odbc-api-reference.md).  
  
 **SQLConfigDataSource** функцию, которая используется для добавления, изменения или удаления источника данных динамически используются следующие ключевые слова.  
  
|Ключевое слово|Описание|  
|-------------|-----------------|  
|COLLATINGSEQUENCE|Последовательность, в которой сортируется поля.<br /><br /> Таким образом задается один и тот же параметр как **упорядочивания последовательности** в диалоговом окне программы установки.|  
|COMPACT_DB|Сжатие данных на файл базы данных. Имеет следующий формат: COMPACT_DB = < путь >< optionaL_sort_order >\<Необязательное ключевое слово ENCRYPT >.<br /><br /> При использовании ключевого слова COMPACT_DB в одной и той же инструкции с ключевым словом DSN, этот драйвер не учитывает ключевое слово DSN. Таким образом сжатие базы данных и указав имя источника данных — это двухэтапный процесс.|  
|КЛЮЧЕВОЕ СЛОВО CREATE_DB КАК|Создает файл базы данных. Имеет следующий формат: ключевое слово CREATE_DB как = < путь >\<порядок optional_sort >< optional_ENCRYPT ключевое слово >, где имя пути — полный путь к базе данных Microsoft Access. Если путь указывает существующую базу данных, будет возвращена ошибка. Порядок сортировки будет как набор вверх в диалоговом окне Новая база данных отображается при нажатии «создать» в диалоговом окне Установка Microsoft Access. Если порядок сортировки не указан, используется Общие.<br /><br /> При использовании ключевого слова ключевое слово CREATE_DB как в одной и той же инструкции с ключевым словом DSN, этот драйвер не учитывает ключевое слово DSN. Таким образом Создание базы данных и указав имя источника данных — это двухэтапный процесс. Если при использовании ключевого слова ключевое слово CREATE_DB как путь к базе данных Microsoft Access, чтобы создать содержит один или несколько пробелов, затем полное имя пути должно быть заключено в двойные кавычки, как показано в следующих примерах:<br /><br /> «C:\PROGRAM FILES\COMMON FILES\ MyAccess.mdb»<br /><br /> «C:\PROGRAM FILES\Access2.mdb»<br /><br /> CREATE_DB=C:\TEMP\test.mdb (без кавычек требуется)|  
|CREATE_SYSDB|Создает файл базы данных системы. Имеет следующий формат: CREATE_SYSDB =\<путь name >\<необязательно порядок сортировки >, где имя пути — это полный путь к базе данных Microsoft Access. Если путь указывает существующую базу данных, будет возвращена ошибка. Порядок сортировки будет как набор вверх в **новую базу данных** диалоговое окно отображается, когда **создать** кнопки в **Установка драйвера ODBC Microsoft Access** диалоговое окно. Если порядок сортировки не указан, используется Общие.|  
|CREATE_V2DB|Создает файл базы данных, которая совместима с Microsoft Access 2.0. Имеет следующий формат: CREATE_V2DB =\<путь name >\<необязательно порядок сортировки >, где имя пути — это полный путь к базе данных Microsoft Access. Если путь указывает существующую базу данных, будет возвращена ошибка. Порядок сортировки будет как набор вверх в диалоговом окне Новая база данных отображается при нажатии «создать» в диалоговом окне Установка Microsoft Access. Если порядок сортировки не указан, используется Общие.<br /><br /> При использовании ключевого слова CREATE_V2DB в одной и той же инструкции с ключевым словом DSN, этот драйвер не учитывает ключевое слово DSN. Таким образом Создание базы данных и указав имя источника данных — это двухэтапный процесс.<br /><br /> При использовании ключевого слова CREATE_V2DB, если путь к базе данных Microsoft Access, чтобы создать содержит один или несколько пробелов, затем полное имя пути должно быть заключено в двойные кавычки, как показано в следующих примерах:<br /><br /> «C:\PROGRAM FILES\COMMON FILES\ MyAccess.mdb»<br /><br /> «C:\PROGRAM FILES\Access2.mdb»<br /><br /> CREATE_V2DB=C:\TEMP\test.mdb (без кавычек требуется)|  
|DBQ|Имя файла базы данных.<br /><br /> Таким образом задается один и тот же параметр как **базы данных** в диалоговом окне программы установки.|  
|ЕГО ЗНАЧЕНИЯ|Строка пути к файлу базы данных.|  
|DESCRIPTION|Описание данных в источнике данных.<br /><br /> Таким образом задается один и тот же параметр как **описание** в диалоговом окне программы установки.|  
|DRIVER|Строка пути к DLL-Библиотека драйвера.|  
|DRIVERID|Целочисленный идентификатор для драйвера.  25 (Microsoft Access)|  
|FIL|Файл типа MS Access для Microsoft Access|  
|IMPLICITCOMMITSYNC|Определяет ли драйвером Microsoft Access выполнения внутренней или неявной фиксации асинхронно. Это значение изначально установлено значение «Да», это означает, что драйвер Microsoft Access будет ожидать фиксации в внутренней/неявной транзакции для завершения.<br /><br /> Значение этого параметра не должно изменяться без тщательного анализа последствий. Дополнительные сведения об этом параметре см. в разделе *Microsoft Jet Database Engine Programmer's Guide*.<br /><br /> Таким образом задается один и тот же параметр как **ImplicitCommitSync** в диалоговом окне программы установки.|  
|MAXBUFFERSIZE|Размер внутреннего буфера, в килобайтах, который используется в Microsoft Access для передачи данных и с диска. Размер буфера по умолчанию — 2048 КБ, (отображается как 2048). Можно использовать любое целочисленное значение, кратным 256. Таким образом задается один и тот же параметр как **размер буфера** в диалоговом окне программы установки.|  
|MAXSCANROWS|Количество строк, проверяемых при задании типа данных столбца на основе существующих данных.<br /><br /> Можно ввести число от 1 до 16 для строки для поиска. По умолчанию используется значение 8; Если он имеет значение 0, просматриваются все строки. (Лежащее за пределами ограничения возвратит ошибку.)<br /><br /> Таким образом задается один и тот же параметр как **строк для просмотра** в диалоговом окне программы установки.|  
|PAGETIMEOUT|Указывает период времени, в миллисекундах, страницы (если не используется), остаются в буфере до их удаления. Значение по умолчанию — 5-десятые доли секунды (0,5 секунд). Обратите внимание на то, что этот параметр применяется ко всем источникам данных, использующие драйвер ODBC.<br /><br /> Таким образом задается один и тот же параметр как **время ожидания страницы** в диалоговом окне программы установки.|  
|PWD|Пароль.|  
|READONLY|Значение TRUE, чтобы сделать файл доступным только для чтения; Значение FALSE, чтобы сделать файл не только для чтения.<br /><br /> Таким образом задается один и тот же параметр как **только для чтения** в диалоговом окне программы установки.|  
|REPAIR_DB|Восстановление базы данных повреждены, ошибки, возникшей во время процесса фиксации.<br /><br /> При использовании ключевого слова REPAIR_DB в одной и той же инструкции с ключевым словом DSN, этот драйвер не учитывает ключевое слово DSN. Таким образом восстановление базы данных и указав имя источника данных — это двухэтапный процесс.|  
|SYSTEMDB|Для драйвера Microsoft Access, пути к файлу базы данных системы.<br /><br /> Таким образом задается один и тот же параметр как **системной базы данных** в диалоговом окне программы установки.|  
|ПОТОКИ|Число фоновых потоков для ядра, которые будут использоваться. Это значение по умолчанию равно 3, но могут быть изменены.<br /><br /> Таким образом задается один и тот же параметр как **потоков** в диалоговом окне программы установки.|  
|UID|Для драйвера Microsoft Access имя пользователя идентификатор используется для имени входа.|  
|USERCOMMITSYNC|Определяет, ли драйвером Microsoft Access будет выполнять пользовательские транзакции асинхронно. Это значение изначально установлено значение «Да», это означает, что драйвер Microsoft Access будет ожидать передачи в пользовательской транзакции для завершения.<br /><br /> Значение этого параметра не должно изменяться без тщательного анализа последствий. Дополнительные сведения об этом параметре см. в разделе *Microsoft Jet Database Engine Programmer's Guide*.<br /><br /> Таким образом задается один и тот же параметр как **UserCommitSync** в диалоговом окне программы установки.|
