# <a name="bookmark-types"></a>Типы закладок
Все закладки в ODBC 3 *.x* закладок переменной длины. Это позволяет первичный ключ или уникальный индекс, связанный с таблицей для использования в качестве закладки. Закладки также можно 32-разрядное значение, которое использовалось в ODBC 2. *x*. Чтобы использовать закладки с курсором, а ODBC 3 *.x* приложение устанавливает атрибут инструкции SQL_ATTR_USE_BOOKMARK SQL_UB_VARIABLE. Автоматически используется закладка переменной длины.  
  
 Приложение может вызвать **SQLColAttribute** с *FieldIdentifier* аргумент значение SQL_DESC_OCTET_LENGTH для получения длины закладки. Поскольку закладка переменной длины может быть значение типа long, приложение нельзя привязать к столбцу 0 Если закладка будет использоваться для многих строк в наборе строк.  
  
 Закладки фиксированной длины, поддерживаются только для обратной совместимости. Если ODBC 2. *x* приложение, которое работает с ODBC 3 *.x* драйвер вызывает **SQLSetStmtOption** с значение SQL_USE_BOOKMARKS SQL_UB_ON, он сопоставлен диспетчера драйверов для SQL_UB_VARIABLE . Используется закладка переменной длины, даже если заполняются только 32 бита его. Если драйвер поддерживает закладки фиксированной длины, он будет поддерживать закладки переменной длины. Если ODBC 3 *.x* приложение, которое работает с ODBC 2. *x* драйвер вызывает **SQLSetStmtAttr** с значение SQL_ATTR_USE_BOOKMARKS SQL_UB_VARIABLE, он сопоставлен диспетчера драйверов для SQL_UB_ON и используется 32-разрядных закладки фиксированной длины. Атрибут инструкции SQL_ATTR_FETCH_BOOKMARK_PTR затем должен указывать на 32-разрядных закладку. Если использовать закладки длиннее 32 бита, например когда первичные ключи используются в качестве закладки, курсор необходимо сопоставить фактические значения для 32-разрядных значений. Например, то удалось построение хэш-таблицу из них. Когда ODBC 3 *.x* приложение, которое работает с ODBC 2. *x* драйвер привязывает закладку, длина буфера должна быть 4.
