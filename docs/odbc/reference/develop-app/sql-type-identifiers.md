# <a name="sql-type-identifiers"></a>Идентификаторы типа SQL
Каждый источник данных определяет собственные типы данных SQL. ODBC определяет идентификаторы типа и описывает общие характеристики типов данных SQL, которые могут быть связаны с каждой идентификатор типа. Это специфические для драйвера сопоставлением каждого типа данных в источнике данных с идентификатором типа SQL ODBC.  
  
 Например SQL_CHAR является идентификатор типа для символьного столбца с фиксированной длиной, обычно от 1 до 254 символов. Эти характеристики соответствуют типу данных CHAR, из множества источников данных SQL. Таким образом когда приложение обнаруживает, что идентификатор типа для столбца является SQL_CHAR, его можно предположить, что, возможно, дело столбца данных типа CHAR. Тем не менее оно должно по-прежнему проверить длину столбца, что он в байтах, — от 1 до 254 символов; драйвер для отличных от SQL источника данных, например, может сопоставить столбец фиксированной длины 500 символов SQL_CHAR или SQL_LONGVARCHAR, так как ни один не точное совпадение.  
  
 ODBC определяет самые разнообразные идентификаторы типа SQL. Тем не менее драйвер не требуется для использования всех этих идентификаторов. Вместо этого он использует только идентификаторы, он должен предоставлять типы данных SQL, поддерживаемые в источнике данных. Если базовый источник данных поддерживает следующие типы данных SQL для какой идентификатора типа не соответствует, драйвер можно определить идентификаторы еще один тип. Дополнительные сведения см. в разделе [типов данных драйвера, типы дескрипторов, типы сведений, диагностические типы и атрибуты](../../../odbc/reference/develop-app/driver-specific-data-types-descriptor-information-diagnostic.md).  
  
 Полное описание идентификаторы типа SQL, см. в разделе [типы данных C](../../../odbc/reference/appendixes/c-data-types.md) в типах данных приложение D:.
