# <a name="sqlgettypeinfo-function"></a>Функция SQLGetTypeInfo
**Соответствие стандартам**  
 Версия была введена: ODBC 1.0 соответствует стандартам: ISO-92  
  
 **Сводка**  
 **SQLGetTypeInfo** возвращает сведения о типах данных, поддерживаемых источником данных. Драйвер возвращает сведения в форме результирующего набора SQL. Типы данных предназначены для использования в инструкциях языка определения данных (DDL).  
  
> [!IMPORTANT]  
>  Приложения должны использовать имена типов, возвращаемых в столбце TYPE_NAME **SQLGetTypeInfo** результирующего набора **ALTER TABLE** и **CREATE TABLE** инструкций. **SQLGetTypeInfo** может возвращать более одной строки с одинаковым значением в столбце DATA_TYPE.  
  
## <a name="syntax"></a>Синтаксис  
  
```  
  
SQLRETURN SQLGetTypeInfo(  
     SQLHSTMT      StatementHandle,  
     SQLSMALLINT   DataType);  
```  
  
## <a name="arguments"></a>Аргументы  
 *StatementHandle*  
 [Вход] Дескриптор инструкции для результирующего набора.  
  
 *Тип данных*  
 [Вход] Тип данных SQL. Это должно быть одно из значений в [типы данных SQL](../../../odbc/reference/appendixes/sql-data-types.md) раздел типы данных приложение D: или к типу данных специфические для драйвера SQL. SQL_ALL_TYPES указывает, что должно возвращаться сведения обо всех типах данных.  
  
## <a name="returns"></a>Возвращает  
 Значение SQL_SUCCESS, SQL_SUCCESS_WITH_INFO, SQL_STILL_EXECUTING, значение SQL_ERROR или SQL_INVALID_HANDLE.

## <a name="comments"></a>Комментарии  
 **SQLGetTypeInfo** возвращает результаты в виде стандартных результирующий набор, упорядоченный по значению DATA_TYPE, а затем по как данные типа аналогичному типу данных ODBC SQL. Типы данных, определенные в источнике данных имеют приоритет над определяемые пользователем типы данных. Таким образом порядок сортировки не обязательно соответствует, но может быть обобщить как DATA_TYPE, во-первых, следуют TYPE_NAME, оба по возрастанию. Предположим, что источник данных определенных типов данных INTEGER и СЧЕТЧИКА, где СЧЕТЧИК заданы с автоматическим приращением, а также был определен, определяемыми пользователем данными типа WHOLENUM. Они будут возвращены в целое число, порядок и WHOLENUM и СЧЕТЧИК, так как WHOLENUM строго соответствует данных ODBC SQL введите SQL_INTEGER, пока тип данных заданы с автоматическим приращением, несмотря на то, что поддерживается источником данных, не наиболее точно соответствуют типу данных ODBC SQL. Сведения о том, как эта информация может использоваться, см. в разделе [инструкции DDL](../../../odbc/reference/develop-app/ddl-statements.md).  
  
 Если *DataType* аргумент указывает тип данных, которое является допустимым для версии ODBC, поддерживаемых драйвером, но не поддерживается драйвером, то возвращается пустой результирующий набор.

> [!NOTE]  
>  **SQLGetTypeInfo** могут не возвращать все типы данных. Например драйвер не могут возвращать определяемые пользователем типы данных. Приложения могут использовать любой допустимый тип данных, независимо от того, он возвращается в виде **SQLGetTypeInfo**. Типы данных, возвращенных **SQLGetTypeInfo** , поддерживаемые источником данных. Они предназначены для использования в инструкции языка определения данных (DDL). Драйверы могут возвращать данные результирующего набора, с помощью типов данных, отличных от типов, возвращенного методом **SQLGetTypeInfo**. При создании результирующего набора для функции каталога, драйвер может использовать тип данных, который не поддерживается источником данных.

|UNSIGNED_ATTRIBUTE (ODBC 2.0)|10|Smallint|Является ли тип данных без знака:<br /><br /> SQL_TRUE, если тип данных не имеет знака.<br /><br /> Значение SQL_FALSE, если тип данных имеет знак.<br /><br /> Если атрибут не применим к типу данных или тип данных не является числом, возвращается значение NULL.|

|AUTO_UNIQUE_VALUE (ODBC 2.0)|12|Smallint|Является ли тип данных автоматическое приращение выполняется:<br /><br /> SQL_TRUE, если тип данных является автоматическим приращением.<br /><br /> Значение SQL_FALSE, если тип данных не является автоматическим приращением.<br /><br /> Если атрибут не применим к типу данных или тип данных не является числом, возвращается значение NULL.<br /><br /> Приложение может вставлять значение в столбца должны иметь этот атрибут, но обычно не удается обновить значения в столбце.<br /><br /> При внесении вставки в столбец автоприращения, уникальное значение вставляется в столбец во время операции вставки. Приращение не определен, но специфического для источника данных. Приложения не должен предполагать, что столбец автоприращения начинается в любой определенный момент или с шагом по любой определенное значение.|  
|LOCAL_TYPE_NAME (ODBC 2.0)|13|Varchar|Локализованная версия имени зависит от источника данных типа данных. Если локализованное имя не поддерживается источником данных, возвращается значение NULL. Это имя предназначено для отображения только, как показано диалоговым окнам.|  
|MINIMUM_SCALE (ODBC 2.0)|14|Smallint|Минимальный масштаб типа данных в источнике данных. Если тип данных имеет фиксированный масштаб, это значение содержится и в столбце MINIMUM_SCALE, и в столбце MAXIMUM_SCALE. Например столбец SQL_TYPE_TIMESTAMP может иметь фиксированный масштаб для долей секунды. Для типов данных, к которым понятие масштаба не применимо, возвращается значение NULL. Дополнительные сведения см. в разделе [размер столбца, десятичных разрядов, длительность октета передачи и отображаемый размер](../../../odbc/reference/appendixes/column-size-decimal-digits-transfer-octet-length-and-display-size.md) в типах данных приложение D:.|  
|MAXIMUM_SCALE (ODBC 2.0)|15|Smallint|Максимальный масштаб типа данных в источнике данных. Для типов данных, к которым понятие масштаба не применимо, возвращается значение NULL. Если максимальный масштаб не определен отдельно в источнике данных, но вместо этого определяется так же, как максимальная точность, этот столбец содержит то же значение, что столбца COLUMN_SIZE. Дополнительные сведения см. в разделе [размер столбца, десятичных разрядов, длительность октета передачи и отображаемый размер](../../../odbc/reference/appendixes/column-size-decimal-digits-transfer-octet-length-and-display-size.md) в типах данных приложение D:.|  
|SQL_DATA_TYPE (ODBC 3.0)|16|smallint NOT NULL|Значение типа данных SQL, так как он отображается в поле SQL_DESC_TYPE дескриптора. Этот столбец является таким же, как столбец DATA_TYPE, за исключением типов данных datetime и интервал.<br /><br /> Для типов данных datetime и интервал SQL_DATA_TYPE поле в результирующем наборе будет возвращать SQL_INTERVAL или SQL_DATETIME и поле SQL_DATETIME_SUB вернет дополнительным кодом для конкретного типа данных interval или даты и времени. (См. в разделе [типы данных приложение D:](../../../odbc/reference/appendixes/appendix-d-data-types.md).)|  
|SQL_DATETIME_SUB (ODBC 3.0)|17|Smallint|Если значение SQL_DATA_TYPE равно SQL_DATETIME или SQL_INTERVAL, этот столбец содержит подкода даты и времени или интервал. Для типов данных, кроме datetime и интервала это поле имеет значение NULL.<br /><br /> Для интервала или типа данных datetime SQL_DATA_TYPE поле в результирующем наборе будет возвращать SQL_INTERVAL или SQL_DATETIME и поле SQL_DATETIME_SUB вернет дополнительным кодом для конкретного типа данных interval или даты и времени. (См. в разделе [типы данных приложение D:](../../../odbc/reference/appendixes/appendix-d-data-types.md).)|  
|NUM_PREC_RADIX (ODBC 3.0)|18|Целочисленный|Если тип данных является приблизительный числовой тип, этот столбец содержит значение 2, чтобы указать, что COLUMN_SIZE указывает на число битов. Для точные числовые типы этот столбец содержит значение 10, чтобы указать, что COLUMN_SIZE указывает общее число десятичных знаков. В противном случае этот столбец содержит значение NULL.|  
|INTERVAL_PRECISION (ODBC 3.0)|19|Smallint|Если тип данных является типом данных интервал, этот столбец содержит значение начальные точности интервала. (См. в разделе [точность типа данных Interval](../../../odbc/reference/appendixes/interval-data-type-precision.md) в типах данных приложение D:.) В противном случае этот столбец содержит значение NULL.|  
  
 Сведения об атрибутах можно применять к типам данных или в определенные столбцы в результирующем наборе. **SQLGetTypeInfo** возвращает сведения о атрибуты, связанные с типами данных. **SQLColAttribute** возвращает сведения об атрибутах, связанных со столбцами в результирующий набор.
